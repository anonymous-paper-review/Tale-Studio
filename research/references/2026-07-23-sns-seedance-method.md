# 레퍼런스 해석 — SNS에서 도는 Seedance 연속성 워크플로 (스킬 3종 실물 분석)

> **한 줄 결론: SNS 방식의 실체는 "시트 한 장을 영상 모델에 넣는 것"이 아니라, ① 캐릭터 마스터 시트로
> 얼굴을 잠그고 ② 콘티 시트를 "한 번의 이미지 생성"으로 만들어 컷 간 일관성을 공짜로 확보한 뒤 셀을
> 잘라 쓰고 ③ 샷마다 시작/끝 프레임 + 텍스트 계약(연속성 바이블·네거티브·결말 상태)을 Seedance의
> 첫 프레임 고정·시드 고정 기능에 넘기는 3단 파이프라인이다.** 우리가 설계한 B1(시작+끝 프레임)과
> C(체이닝)가 생성 역학의 정답에 근접해 있었고, 우리가 놓친 것은 **텍스트 계약 계층**과 **"시트에서
> 프레임을 수확한다"는 발상**이다.
>
> 해석일 2026-07-23 · 원본: `~/Downloads/ref/fable_prompt_sheet/` (스킬 zip 5종 + 시트 실물 PNG 30여 장)

## 1. 이 방식은 스킬 3종 파이프라인이다

**① 마스터 시트 (master-sheet v1/v2)** — 캐릭터/제품의 일관성 레퍼런스 시트를 이미지 생성기로 한 장
뽑는다. v2 실물(첨부 캐릭터 시트): 3:2 가로 그리드에 정면(대형)·3/4·측면·전신·표정 3컷, 텍스트는 패널
헤더뿐. 주목할 문장이 스킬 안에 있다:

> "Element 락은 별도 단계: **깨끗한 단일 히어로/정면 1컷**을 락 소스로 권장 (멀티패널 시트 통째보다
> 생성 전이가 안정적)."

— **우리가 실측으로 도달한 결론(정면 1장 = 신원 정본)과 정확히 같은 문장**이 이 동네의 검증된 교훈으로
이미 박혀 있다.

**② 콘티 시트 (storyboard v1/v2)** — v1은 광고 한 편 전체를 12컷 격자 한 장으로(실물: "옥수수염차
15초/12컷" — 셀마다 썸네일+ACTION/CAMERA/DIALOGUE), v2는 **한 컷을 6샷(S1~S6)으로 쪼개 샷마다
START/END 프레임 묘사**까지 박은 정밀 분해 시트다. 핵심 트릭: 콘티 시트가 "한 번의 이미지 생성"이라
**12컷 썸네일끼리 인물·세계 일관성이 자동으로 유지**된다 — 컷별로 따로 그리면 깨질 일관성을 시트
생성이 공짜로 잡아주고, 그 셀들을 잘라 샷별 프레임/참조로 쓴다. v2 스킬의 마지막 줄:

> "Seedance 2.0로 넘길 때 각 샷의 START→END를 start_image/end_image 체이닝 소스로 그대로 활용."

— **우리 B1(시작+끝 프레임 쌍)·C(체이닝)와 같은 생성 역학**이다.

**③ 연속성 빌더 (seedance-continuity-builder v5.3)** — 영상 모델에 붙여넣는 **텍스트 계약**을 만든다:
- **Continuity Bible**: 세계·인물(의상 LOCKED)·시간(빛 방향까지)·공간(180° 라인, 3줄 정신 지도)·
  시그니처 소품·모티프를 먼저 잠그고 모든 샷이 참조
- **네거티브 프롬프트 배터리**: 의상 플리커·그림자 방향 뒤집힘·낮밤 점프·다중 얼굴·플라스틱 피부 등
  **드리프트를 문장으로 금지**
- **결말 상태(Ending State)**: 세그먼트 끝의 정확한 자세·카메라·조명을 적어 다음 세그먼트의 시작
  계약으로 넘김 (우리 "동작 이어받기"의 텍스트판)
- **카메라 4요소** 〈렌즈·앵글/구도·무브·광학〉 + 직전 샷과 앵글 중복 금지
- 플랫폼 기능 활용: **첫 프레임 고정(픽셀 단위)** · **시드 고정(시리즈 일관성)** · 참조 이미지 ≤9장 ·
  15초 세그먼트 체인(연장 기능)

## 2. 우리 설계(입력 포맷 실험 A/B1/B2/C)와의 대조

| | SNS 방식 | 우리 설계 | 판정 |
|---|---|---|---|
| 신원 잠금 | 마스터 시트 → 단일 정면 1컷 락 | 레퍼런스 정면 1장 정본 (실측 검증) | **일치** — 같은 결론 |
| 생성 역학 | 샷별 START/END + 첫 프레임 고정 + 체이닝 | B1(시작+끝) + C(체이닝) | **일치** — B1·C가 정답 계열 |
| 시트의 용법 | **설계 시각화 + 프레임 수확** (한 생성 안 12컷 = 일관성 공짜) | B2 = 시트를 영상 모델에 통째 입력 (가설) | **다름** — 우리 B2는 실제 SNS 용법이 아니었음 |
| 텍스트 계약 | Bible·네거티브 배터리·결말 상태 | 앵커 접두 문장 하나 | **우리가 빈약** — 가장 큰 수확 |
| 시드 | 영상 시드 고정(시리즈) | 이미지 시드만 고정 | 우리가 누락 |
| 카메라 표기 | 4요소 조합 + 반복 금지 | 사이즈·무브 위주 | 우리가 얇음 |
| 오디오 | 3레이어 설계 내장 | 범위 밖(무음) | 차후 |
| 콘티 정답지·정량 채점 | 없음 (감으로 평가) | 원본 프레임 대조 + 임의 무브/붕괴 카운트 | **우리가 우위** |
| 자동화 | 수작업 UI 붙여넣기 | writer 파이프라인 연동 목표 | 우리가 우위 |

## 3. 실험 설계에 반영한 것 (입력 포맷 실험 v2)

1. **전 팔 공통으로 "텍스트 계약" 추가** — 앵커 한 문장 → 연속성 바이블(의상 LOCKED·빛 방향·180°
   라인) + 드리프트 네거티브 배터리 + 샷별 결말 상태. 팔 간 차이는 여전히 "이미지를 어떻게 주느냐"로
   유지(통제 변수 원칙).
2. **B2 재정의** — "시트 통째 입력"(원안)은 β 시도 1회로 강등해 오작동 여부만 기록하고, B2의 본체는
   **"시트 생성 → 셀 수확"**으로 교체: 6샷의 시작 프레임을 콘티 시트 한 장 생성으로 만들어(셀 간
   일관성 공짜 가설) 잘라 쓰고, 이후는 B1과 동일하게 생성. B1(샷별 개별 생성)과의 차이 = 프레임
   제작법이 되므로 "이미지 계층 일관성"을 가르는 팔이 된다.
3. **선행 스크리닝에 Seedance(첫 프레임 고정) 필수 포함** — fal에 있는 ByteDance 계열이 유일하게
   "픽셀 단위 첫 프레임 고정"을 명시하는 후보. 영상 시드 고정 지원 여부도 스크리닝 항목에 추가.
4. **카메라 4요소 표기**는 콘티 정답지가 이미 고정 카메라라 이번 실험엔 영향 없음 — writer 본체
   개선 후보로 개선 백로그에 기록(I9 후보).
