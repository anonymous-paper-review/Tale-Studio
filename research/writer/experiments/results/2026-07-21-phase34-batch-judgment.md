## 2. 판정끼리의 상호작용 (같이 결정하면 좋은 것)

- **E9 + E13은 같은 방향의 구조 결정** (콜 병합으로 12→7콜 로드맵의 몸통): 둘 다 채택하면 파이프라인이
  S(1콜)→V(축소)로 크게 줄지만, 정식 구현+재검증이라는 같은 종류의 후속 작업이 두 건 생긴다. 하나만
  먼저 간다면 리스크가 작은 쪽은 E9 (기존 Compact 분기가 이미 프로덕션 코드).
- **E8 채택은 E6 삭제와 충돌하지 않는다** — E8은 v1→V3 배선, E6은 midPreview seed. 서로 다른 배선이며
  실험도 분리 측정됨.
- **E11의 "언어 명시" 는 판정과 무관하게 필요** — 한국어 정본을 골라도 명시 지시는 넣어야 한다
  (현행은 모델이 언어를 임의 선택). 이것만은 W그룹(청소)급 후속으로 분리 가능.

## 3. 판정 후 실행 순서 제안 (참고용)

1. 즉시형 (판정만 하면 반나절 내): E6 삭제 / E8 배선 1줄 / E11 언어 명시 지시
2. 재검증 동반형: E9 (샷 예산 힌트 확장 + 이관 설계) → E13 (정식 스테이지 구현 + 후처리 이관 + 재검증)
3. E8 채택 시 선행 확인 1회: 실제 v2 체인에서 palette_emphasis ⊂ 전역 팔레트 준수율

## 4. 이 문서로 판정하는 법

각 행에 대해 3택: **채택 / 미채택 / 조건 변경 후 재측정**. 판정 결과는 이 파일 하단이 아니라
각 결과 파일 + 계획 문서 상태표에 기록한다 (진실은 results/ 규칙 유지).
