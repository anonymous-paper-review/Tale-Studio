# 판정 대기 실험 5건 — 한눈 비교표 (일괄 판정용, 2026-07-21)

> **용도: 제품 오너가 5건을 한 번에 판정하는 문서.** 5건 전부 측정은 끝났고 제품 동작은 아직 아무것도
> 바꾸지 않았다 — 판정 전까지 실서비스는 이 실험들의 영향을 전혀 받지 않는다.
> 각 실험의 입력·출력 원문과 자세한 근거는 개별 결과 문서(각 행에 링크)가 진실이다.

## 2. 판정끼리의 상호작용 (같이 결정하면 좋은 것)

- **E9와 E13은 같은 방향의 구조 결정**이다 (LLM 호출 12회→7회로 줄이는 로드맵의 몸통). 둘 다 채택하면
  파이프라인이 크게 줄지만, "정식 구현 후 재검증"이라는 같은 종류의 후속 작업이 두 건 생긴다. 하나만
  먼저 간다면 리스크가 작은 쪽은 E9 — 합쳐진 경로가 이미 제품 코드에 존재한다(짧은 영상용으로).
- **E8 채택과 E6 삭제는 충돌하지 않는다** — 서로 다른 연결선이고 실험도 분리 측정했다.
- **E11의 "언어 명시"는 판정과 무관하게 필요하다** — 한국어 정본을 골라도 명시 지시는 넣어야 한다
  (지금은 모델이 언어를 임의 선택). 이것만은 실험 없이 적용 가능한 청소성 수정으로 분리할 수 있다.

## 3. 판정 후 실행 순서 제안 (참고용)

1. **즉시형** (판정만 하면 반나절 내): E6 삭제 / E8 연결 1줄 / E11 언어 명시
2. **재검증 동반형**: E9 (샷 개수 힌트 확장 + 참조 이관 설계) → E13 (정식 구현 + 뒷정리 이관 + 재검증)
3. E8 채택 시 선행 확인 1회: 실제 색 계획 단계와 연결한 상태에서 "전체 색 목록 안 변주" 준수율

## 4. 이 문서로 판정하는 법

각 행에 대해 3택: **채택 / 미채택 / 조건 변경 후 재측정**. 판정 결과는 이 파일이 아니라
각 결과 문서 + 계획 문서 상태표에 기록한다 (진실은 개별 결과 문서 규칙 유지).

## 5. 판정 기록 (2026-07-22, 제품 오너)

5건 중 4건에 판정이 나왔다. 공통 방향: 즉시 채택/기각이 아니라 **"조건을 바꿔 한 번 더 재보라"**.

| 실험 | 판정 | 후속 |
|---|---|---|
| E6 밑그림 | **조건 변경 후 재측정** — 밑그림 프롬프트는 다듬어진 적이 없으니 최적화부터 하고 나서 삭제 여부를 정한다 | E6b (측정 완료 — 판정 대기) |
| E8 색·조명 배선 | **조건 변경 후 재측정** — "작품 전체 색 계획 안에서의 변주인지" 판별 기준을 세우고 진짜 세계 팔레트 체인으로 재확인 | E8b (측정 완료 — 이탈 0%·색온도 진행 증폭, 단 다양성 효과는 소멸) |
| E9 촬영 계획 흡수 | **조건 변경 후 재측정** — "씬을 상세히 쓰고 샷은 집행"안 vs "씬과 샷을 거의 같은 순간에 생성"안을 비교. 목표 분량·샷 길이 충족을 최우선 지표로 | E9b (측정 완료 — 최우선 3지표 동률, 갈림길은 통제·속도 vs 연출 풍부함) |
| E13 구조+장면 병합 | **조건부 진행** — 선행 3건(정식 구현·표본 확대·체크포인트 재설계)을 상세히 수행한 뒤 재검증 | E13b (측정 완료 — 사전 기준 4/4 통과, **채택 상신**) |
| E11 렌더 문장 언어 | **판정 미기재** — 이번 일괄 판정에서 언급되지 않아 대기 유지. 권고(영어 정본 + 언어 명시 지시)는 그대로 유효 | 판정 대기 |

판정 대기 5건 외 항목에 대한 오너 코멘트도 같은 날 기록됐다 — 상세는 계획 문서 §6(개선 목록)과 각 결과 문서 머리말 참조. 요지:

- **E0a**: 실험은 인정하되 적용 범위 한계 지적 — 프로브가 "정답이 명시된 입력"이라, 실사용자의 모호한 입력에서 의도를 파악해 구조를 고르는지는 미검증 (별도 실험 E0c 후보 등록, 실행 여부는 오너 판정 대기).
- **E3a**: 씬 시간과 행동·샷 밀도의 정합 문제를 개선 목록에 등록 (계획 문서 I1).
- **D1**: 감사 대상이 실험산 데이터 위주 + 감사 스크립트 일부가 LLM 작성 — 정답 기준으로는 절반 신뢰, 방향 지표로만 쓰고 재감사는 예산상 보류.
- **R1**: 통과 유지. 부수 관찰 2건(샷 2초 하한 미검증·모션 문구 글자 수 이탈)은 개선 목록 I2·I3으로 이동. 로컬 log 파이프라인은 폐기됐고 운영 산출물의 진실원은 Supabase DB 단일임이 명시됨.
- **E1**: 이해·수용 (추가 액션 없음).
- **E5**: 실험 유효하나 C1 검수기 자체가 현재 운영에서 꺼져 있음(writer 속도 문제) — 재정의판은 재가동 시점의 사양으로 보존.
- **E12b**: 채택 유지 + 파생 질문 — "facet을 만드는 시점에 최종 산출물도 같이 만들어 내보내면 되지 않나?" → 코드 조립 방식과 속도·품질 비교 실험 E12c로 등록 (설계 중).
