## 요약 (쉬운 말)

지금은 "씬을 어떻게 찍을지(V3)"와 "샷을 어떻게 나눌지(decoupage)"를 AI에게 두 번 시킨다. 짧은 영상용
코드에는 이미 한 번에 시키는 경로(Compact)가 있어서 그걸 전 구간 후보로 놓고 비교했다. 결과: **한 번에
시키면 시간이 약 40% 절약**되고 샷 규율(길이 물리)은 동등한데, **샷을 더 많이 만든다**(kishoten 평균
16.5→20.5개 — 샷 수는 곧 하류 생성 비용). 그리고 흥미로운 실측: 2단을 유지해도 decoupage는 V3가 정한
샷 수 목표를 어차피 +3~4개씩 초과하고 있었다 — 2단의 명분인 "권위 사슬"이 실제로는 반쪽으로만 작동 중.

## 2. 핵심 관찰

- **권위 사슬은 이미 반쪽**: 2단에서도 decoupage는 V3의 shot_count_target을 지키지 않는다(4/4 run 초과).
  "V3가 정하고 decoupage가 따른다"는 배선 서사가 실측과 다름 — 흡수의 명분(모순 해소)이 실증됨.
- **흡수의 대가는 샷 인플레**: V3 플랜이 있으면 added가 절제된다(kishoten 2.5 vs 5.5). 샷 수는 하류
  이미지·클립 생성 비용에 직결 — 흡수 채택 시 **씬당 샷 예산 힌트**(E3b 대표 모드에서 이미 구현한
  메커니즘)를 정직 대역에도 확장하는 것이 자연스러운 동반 조치.
- B팔은 scene_plans를 만들지 않음 — V3 산출 소비처(V4 일반 모드 디시플린, validator)의 이관 설계가
  채택의 실제 작업량.

## 3. 권고 (판정은 보류)

**조건부 흡수 우세** — 시간 40%·콜 1개 절감은 확실하고 권위 사슬 명분은 이미 깨져 있음. 단 두 조건 동반:
① 씬당 샷 예산 힌트로 샷 인플레 통제(기존 메커니즘 확장 — 신규 개발 아님) ② V4 일반 모드·validator의
디시플린 소스 이관 설계. 장편(D6~D7)은 이미 대표 모드 힌트가 있어 영향 작음. 유지를 고르면 현행 비용
그대로 + target 미준수 모순은 별도 수선 필요(사슬을 실제로 강제하거나 명분을 지우거나).
