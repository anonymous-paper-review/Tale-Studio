## 요약 (쉬운 말)

v1 단계는 "막이 진행될수록 화면의 색·조명이 어떻게 변해야 하는가"를 매번 설계하는데, 정작 그 설계를
받아 쓰는 곳(V3)에 전달되지 않고 버려지고 있었다(죽은 배선). 이번에 전달해 봤더니 — **버려지던 설계가
실제로 화면 계획을 바꾼다**: 막 사이 조명 온도 진행폭이 약 2배로 벌어지고(도입은 차갑게 → 결말은
따뜻하게 식는 진행이 뚜렷해짐), 막마다 쓰는 색 종류도 3색 고정에서 6~11색으로 분화됐다. 즉 v1은
"무가치해서 버려진 것"이 아니라 "배선이 끊겨 있었을 뿐"이다. 이제 선택지는 배선을 잇거나(막별 시각
진행 획득), v1을 지우거나(콜 1개 ~7초 절감) 둘 중 하나다.

## 1. 결과 (8/8 성공, 재시도 0)

| 지표 | ctl (현행 무배선) | arc (주입) |
|---|---|---|
| 막간 색온도 격차 (막 평균 K의 최대−최소) | 750~1900K (평균 ~1350K) | **1850~3250K (평균 ~2650K, 약 2배)** |
| 팔레트 고유수 (전체) | 3 (스텁 전역 팔레트 반복) | **6~11 (막별 분화)** |
| 진행 방향성 | 있음 (V3 자체도 완만한 진행을 만듦) | 동일 방향에 진폭 확대 |
| v1 콜 비용 | 6~7s (현행도 이미 지불 중 — 산출만 버림) | 동일 |
| V3 이슈 수 | 9~14 | 9~14 (동등 — 주입이 검증 위반을 늘리지 않음) |

- 관찰: v1 산출의 energy 곡선은 4막 구조에서 8/8 run 전부 [low/rising/high/falling]로 결정적 — 구조가
  같으면 v1의 거시 형태는 사실상 상수이고, 값은 팔레트·조명 문구에서 갈린다.

## 2. 주의 (채택 전 확인 항목)

- **팔레트 분화 vs 전역 팔레트 준수의 긴장**: arc팔의 색 다양성 증가가 전역 팔레트(v2) 안의 변주인지
  이탈인지 이번 측정으로는 판별 불가 — V축 스텁(3색 고정 팔레트)을 썼기 때문. 정식 채택 전 실제 v2
  체인으로 "palette_emphasis ⊂ 전역 팔레트" 준수율 확인 필요.
- v1 프롬프트의 "≤10분 전제" 문구는 어느 판정이든 제거 대상 (감사 1.6 — 장편 표면 대역과 모순).

## 3. 권고 (판정은 보류)

**배선 복원 우세** — 죽은 배선의 산출이 실제로 유효한 신호였음이 실증됐고, 복원의 추가 비용은 0
(v1은 이미 매 run 실행·과금 중이며 V3 프롬프트에 블록 하나 추가가 전부 — 이번 실험 코드가 그대로
구현체). 제거를 고르면 ~7s/run 절감하지만 막별 시각 진행 능력을 버린다. 판정 축은 기술이 아니라
제품: **막별 시각 진행(도입-절정-여운의 화면 톤 변화)이 결과물 가치인가?** — 채택 시 위 팔레트 준수
확인 1회 + 호출부에서 파라미터 전달(1줄)로 완료.
