# E1 — 저작 지시문 맨 앞에 확정값 요약을 붙이면 사용자가 정한 내용을 더 잘 지키는지 검사

> **한 줄 결론: 미채택 — 바닥 효과.** 지금 지시문이 이미 사용자 확정값 위반을 거의 0으로 만들고
> 있어서, 요약을 얹어도 줄어들 위반이 없었다 — 16개 비교(합계 32번 실행) 전부에서 무승부.
>
> 실행일 2026-07-21 · 판정 완료 ✅ · 기술 재현 정보는 맨 아래 부록.

## 1. 무엇을 의심했나

파이프라인의 장면 나누기·샷 설계 단계는 producer가 이미 확정한 사실 — 스토리에 적힌 사건·인물·
장소·소품·결말, 캐스트/로케이션의 정확한 이름(id) — 을 그대로 지키면서 그 위의 디테일만 채워야 한다.
이 확정 사실들은 지금 지시문 안에서 [기존 캐스트]/[기존 로케이션] 목록이나 스토리 본문 속에 섞여서
등장한다. 걱정은: 지시문이 길어지고 뒤로 갈수록, 모델이 이 확정 사실을 놓치거나(예: 유저가 정한
소품이 사라짐, 정한 결말이 안 지켜짐) 근거 없이 새 인물을 만들어 넣거나(발명 남발) 하지 않을까 하는
것.

의심이 맞다면 해법은 이렇다 — 지시문 맨 앞에 "이건 절대 못 바꾸는 확정값 / 이건 네가 채울 빈칸 /
빈칸은 이런 원칙으로 채워라"라는 3단 요약을 표준 서두로 못박아 붙이면, 확정값 이탈이 줄어들 것이다.
이걸 검증하는 게 이 실험이다.

## 2. 무엇을 넣었나 — 입력 원문

일부러 "지켜야 할 것"을 잔뜩 심어둔 함정 이야기를 썼다 — 고정된 소품(노란 우산), 고정된 마지막 장면
(문고리에 걸린 우산 클로즈업), 뒤집힌 전제(비 오는 날에만 문을 여는 꽃집 — 유저가 만든 차별점),
그리고 표시 이름과 다른 내부 id를 가진 인물·장소(예: 인물 이름은 "연희 할머니"지만 내부 id는
`grandma_yeonhee`)까지 갖췄다:

> 비 오는 날에만 문을 여는 골목 꽃집. 연희 할머니는 빗소리가 들리면 셔터를 올리고 창가에 가장 밝은
> 꽃들을 내놓는다. 손자 민호가 노란 우산을 쓰고 달려와 가게 앞 화분들을 처마 밑으로 옮긴다. 비가
> 그치자 할머니는 천천히 셔터를 내리고, 민호는 노란 우산을 가게 문고리에 걸어둔다. 마지막 컷은 문고리에
> 걸린 노란 우산 클로즈업.

이후 제품 오너 지시("최소 10개 이상의 플롯으로 검증하라")에 따라 같은 함정 설계(고정 소품·고정
결말·표시 이름과 다른 id)를 호러·로맨스·스릴러·코미디·가족드라마·SF·판타지·스포츠·뮤직비디오·푸드
광고 10개 장르로 확장했다. 예를 들어 호러 소재는:

> 폐가 탐험 유튜버 지우가 낡은 저택에 들어간다. 삐걱이는 계단, 먼지 낀 거울, 갑자기 닫히는 문. 거울에
> 비친 흰 소복의 여인이 순간적으로 나타나고, 지우는 카메라를 떨어뜨리고 도망친다. 마지막 컷은 바닥에
> 떨어진 카메라 화면 속에서 여인이 카메라를 향해 천천히 다가오는 장면.

코미디 소재는:

> 막내 사원 봄이가 부장 몰래 탕비실에서 생일 케이크를 준비한다. 까치발, 숨죽인 초 꽂기, 그때 부장이
> 문을 벌컥 연다 — 봄이는 케이크를 등 뒤로 숨기고 어색하게 웃는다. 부장이 "내 생일 아닌데?"라고
> 하자 전 직원이 뒤에서 폭죽을 터뜨린다. 오늘은 봄이의 생일이었다. 마지막 컷은 크림 묻은 봄이 얼굴
> 클로즈업.

10개 장르 전부 같은 설계 원칙(고정 소품·고정 결말·표시 이름≠내부 id) + 해당 장르의 대표적인 클리셰
(점프스케어, 오해로 인한 갈등, 서프라이즈 등)를 의도적으로 하나씩 포함시켰다.

## 3. 어떻게 실험했나

- 현행(A) = 지금 쓰는 지시문 그대로. 제안(B) = 지시문 맨 앞에 3단 요약 서두를 붙인 버전. 실제로
  주입된 서두는 이렇게 생겼다:

  > [불변 입력 — 변경·재해석 금지]
  > - 유저 스토리에 명시된 사건·인물·장소·소품·결말은 원장이다: 빼거나 다른 것으로 바꾸지 않는다.
  > - runtime 45초, format vertical_9:16, 장르 advertisement/brand_film는 확정값이다.
  > - [기존 캐스트]의 slug와 [기존 로케이션]의 id는 참조 키다 — 글자 그대로 쓴다.
  >
  > [네가 채울 빈칸 — 이 스테이지의 저작 범위]
  > - 씬 분해(경계·순서), 씬별 시간 배분, 감정 비트, 정보 비대칭, scene_actions.
  >
  > [채움 원칙]
  > - 유저가 정하지 않은 모호한 디테일은 이 장르에서 관객이 기대하는 가장 전통적인 선택(관습
  >   디폴트)으로 조용히 채운다.
  > - 관습에서 벗어나는 발명은 유저 입력에 이미 있는 차별점을 증폭할 때만 한다.
  > - 새 인물·새 장소 추가는 기존 것으로 전개가 불가능할 때만 한다.

- 꽃집 이야기와 별도의 광고 소재를 각 3번씩 현행/제안으로 돌리고(본배터리), 이어서 10개 장르를 각
  1번씩 돌렸다(브레드스 배터리, 제품 오너 지시로 추가).
- **판정 기준은 실행 전에 미리 정했다**: 제안 쪽의 위반 합계가 현행보다 적고, 동시에 아래에서 말한
  "지켜야 할 차별점"의 생존율이 현행 이상이며, 부수 지표(줄거리 이탈률·막 커버리지)가 악화되지
  않으면 채택. **둘이 똑같으면(동률) 미채택.**
- 위반은 사람이 읽고 판단하지 않고 전부 코드로 결정론적으로 집계했다: 확정 로케이션 id를 벗어났는지,
  근거 없이 새 인물을 만들었는지, 확정 캐스트가 중간에 사라졌는지, 샷 설계 단계에서 모르는 인물
  id가 등장하는지, 그리고 "노란 우산"류 차별점 5가지가 끝까지 살아남는지.

## 4. 무엇이 나왔나 — 출력 원문

**본배터리(광고·꽃집 × 현행/제안 × 3번, 12개 체인, 에러 0) — 완전 동률.**

| 지표 | 광고/현행 | 광고/제안 | 꽃집/현행 | 꽃집/제안 |
|---|---|---|---|---|
| 확정값 위반 합계 (로케이션+발명인물+캐스트누락+미지인물) | 0 | 0 | 0 | 0 |
| 차별점 생존 (5가지 전부, 3번 중) | — | — | 5/5 전부 3/3 | 5/5 전부 3/3 |
| 줄거리 이탈률 평균 | 0.0% | 0.0% | 0.0% | 0.0% |
| 막 커버리지 위반 | 0 | 0 | 0 | 0 |

꽃집 이야기의 제안(3단 요약 서두를 붙인 쪽) 1번째 실행 마지막 장면 출력은 이렇다:

> 민호가 자신이 쓰고 있던 노란 우산을 닫아 가게 문고리에 조용히 걸어둔다
> 굳게 닫힌 셔터 앞, 문고리에 걸려 있는 노란 우산을 클로즈업하며 마무리된다

유저가 스토리에 지정한 "마지막 컷은 문고리에 걸린 노란 우산 클로즈업"이 글자 그대로 지켜졌다. 그런데
**요약 서두가 없는 현행 쪽도** 1번째 실행에서 똑같이 지켜졌다:

> 민호가 쓰고 있던 노란 우산을 접어 닫힌 꽃집 문고리에 조심스레 걸어둔다
> 문고리에 걸린 노란 우산이 바람에 살짝 흔들리는 모습이 클로즈업된다

즉 서두를 붙이기 전부터 이미 지켜지고 있었다 — 우산 등장 3/3, "노란 우산" 표기 3/3, 마지막 씬 구성
3/3, 비 전제 유지 3/3, 샷 설계 단계까지 우산이 살아남는 것 3/3 — 5가지 차별점 모두 양쪽에서
동일하게 100% 생존했다.

(각주: 광고/제안 3번째 실행에서 모델 원본 응답에 잉여 `}` 문자가 섞여 엄격 파싱이 한 번 실패했으나,
정규식 재추출로 확인한 결과 위반은 0이었다 — 실제 파이프라인이 쓰는 관용적 파서는 정상 처리했으므로
채점 과정에서만 생긴 잡음이다.)

**브레드스 배터리(10개 장르 × 현행/제안, 20번 실행, 에러 0).**

| 합계 (10쌍) | 현행 | 제안 (3단 요약) |
|---|---|---|
| 확정 로케이션 id 위반 | 0 | 0 |
| 근거 없는 인물 발명 | 0 | 0* |
| 확정 캐스트 누락 | 0 | 0 |
| 고정 소품 생존 | 9/10** | 10/10 |
| 고정 결말 생존 | 10/10 | 10/10 |
| 줄거리 이탈률 / 막 커버리지 위반 | 전부 0.0% / 0건 | 동일 |

- \* 1차 채점은 코미디 소재의 제안 결과에서 "office_coworkers"라는 인물이 근거 없이 발명됐다고
  표시했으나, 실제로 확인해보니 이름이 "팀원들"인 이 인물은 스토리 본문의 "전 직원"을 그대로 구현한
  것이었다 — 채점 스크립트의 허용 단어 목록에 "직원/동료/사원"은 있었지만 "팀원"이 빠져 있었던
  채점 오류였다. 교정하면 발명 0.
- \*\* SF 소재의 현행 쪽에서 소품 낱말("신호")이 딱 1번 출력에 등장하지 않았는데, 이건 표현이
  느슨한 단어 하나가 우연히 빠진 노이즈 수준이라 "제안이 더 낫다"는 증거로는 채택하지 않았다.

## 5. 결론과 후속

- **최종 미채택** — 본배터리 6쌍 + 브레드스 10쌍 = **16개 비교(합계 32번 실행) 전부에서 확정값
  위반 0, 차별점 생존도 동률**로 끝났다.
- 원인은 실측으로 명확하다: 지금 지시문에 이미 있는 국소 규칙 — "기존 캐스트는 주어진 이름 그대로
  쓴다", "기존 장소는 주어진 id 그대로 쓴다" — 가 3단 요약 서두가 하려던 일을 이미 다 하고 있었다.
  요약 서두는 같은 진실을 한 번 더 말한 것뿐이라, 지금의 압력 수준에서는 추가 효과가 없다.
- 조치: 이 3단 요약 서두를 켜는 실험용 스위치(환경변수 게이트)는 **제거**한다 — 죽은 코드를 남겨두지
  않는다는 원칙에 따른 것이며, 서두 전문과 실험 결과는 이 문서와 git 이력에 보존한다.
- 이 실험이 검증하려던 아이디어(확정값을 지키게 만들자)는 "서두 형식을 요구하는 규칙"이 아니라
  "프롬프트를 다시 쓸 때 빠뜨리면 안 되는 보존 목록"으로 다시 정의했다 — 앞으로 이 파이프라인 단계의
  지시문을 통째로 재작성하는 작업을 할 때, 지금 국소 규칙들이 하는 일을 유실하지 않도록 가드로 남긴다.
- 다시 검증할 조건: 사용할 모델을 바꿀 때(바닥 효과가 지금 모델의 능력 때문일 수 있어서) 또는 실제
  서비스에서 확정값 위반이 관측될 때.
- 부수 소득: 이번에 만든 10개 장르 함정 프리셋은 이후 다른 실험(프롬프트 재작성 검증 등)에서도
  재사용할 공용 입력 자산이 됐다.
