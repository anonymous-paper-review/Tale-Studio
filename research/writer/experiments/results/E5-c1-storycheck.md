## 1. 가설·판정 기준 (사전 확정)

- 가설 ⓐ (P5 충돌): C1은 의도된 관습(클리셰=명세)을 감점한다.
- 가설 ⓑ (검출력 불명): 진짜 결함(인과 붕괴·핍진성 붕괴·차별점 실종)을 잡는지 모른다.
- 판정 매트릭스: 검출력 없음(주입 4 run 중 2+ 미검출) = 삭제 / 검출력 + 오탐(정상 5건 중 2+에
  클리셰 WARNING+) = P5-정합 재정의 후 재측정 / 검출력 + 무오탐 = 유지.
- 교란 통제: 하네스 캐스트는 personality 빈 스텁 — 판정은 (정상 vs 변이) 차분과 클리셰 카테고리(스텁 무관)만.

## 3. 재정의 내용 (c_validation_1.ts 프롬프트 교체)

- 클리셰 감지 항목 삭제 + 검사 원칙 신설: "관습적·전통적 선택은 결함이 아니다 — 관습 디폴트는 명세"(독트린 §0-5).
- **장르 계약 조항**: 광고의 제품 매직 모먼트는 관습이지 DEM이 아니다. DEM 판정은 장르 계약과 무관한
  외부 우연(셋업 없이 곤경을 대신 해소)에만 — 1차 CRITICAL 오탐(a2·a4)의 직접 교정.
- **차별점 보존 검사 신설** (category: `differentiator`, 타입 유니언에 추가): NS 명시 고유 요소의 실종/대체 —
  1차 mut1에서 모델이 자발적으로 감지한 행동("glowing shoes 실종" 지목)을 정식 항목으로 승격. 독트린 P5의
  "감점 대상 (b) 유저 차별점 실종"의 구현.
- cliche_count는 참고 카운트로 유지(스키마 불변) — 이슈 생성 금지 명시.
- 판별력 시험 설계: 재정의판은 **신발(제품·장르 계약 안)은 통과시키고 트럭(외부 우연)은 계속 잡아야** 한다 —
  같은 DEM 패턴의 관습/비관습 양면을 가르는지가 재측정의 핵심.

## 4. 재측정 (재정의판) — 판정 기준 (사전 확정)

- A′ (정상 4셀): 클리셰 카테고리 이슈 0건 **그리고** 제품 매직 모먼트 CRITICAL 0건 (passed=true 4/4 기대).
- B′ (변이 4런): CRITICAL ≥1 유지 4/4 (mut1 트럭/차별점 실종, mut2 무인과 전환).
- 충족 = 재정의판 채택 커밋. A′ 미충족 = 프롬프트 재수정 1회 후 재재측정. B′ 미충족 = 재정의가 검출력을
  죽인 것 — 원인 분석 후 존폐 재론.

우연"이라고 논거를 명시. 신설한 differentiator 카테고리도 4/4에서 발화 ("제품이 수행할 차별점 역할을

### 최종 판정: **재정의판 채택** (사전 기준 A′·B′ 전부 충족)

**관찰 (a2 셀)**: 재정의판의 유일한 정상-대역 CRITICAL은 shorts e3b2의 differentiator — "제품이 광고
이미지로만 등장, 섭취 없이 각성". 그런데 이건 **유저 스토리 원문 자체의 전제**다(입력: "광고 한 줄에
눈이 번쩍"). C1은 시그니처상 유저 원문(PipelineInput)을 받지 않아 NS만 보고 판단하므로, 유저가 의도한
전제를 차별점 실종으로 볼 수 있다 — 원장 위반의 역방향. 후속: C1 시그니처에 유저 스토리 전달 검토
(현재는 default-skip이라 실해 없음; default-on 논의 전 필수 선행).

## 4.5 브레드스 확장 (E5x — 10장르 × 재정의판 C1 각 1 run, 집계 `../tools/e5x_score.mjs`)

제품 오너 지시("최소 10개 이상의 플롯")로 장르 일반화를 측정. 프리셋 10종에는 구판이 감점하던 관습을
**일부러 심었다**: 점프스케어(호러)·오해 재회(로맨스)·1초 전 해제(스릴러)·서프라이즈 반전(코미디)·
무뚝뚝한 아버지(가족)·선택받은 자(판타지)·역전 슬로모(스포츠)·첫눈 몽타주(MV)·김 클로즈업(푸드).

- **클리셰 카테고리 이슈 0/10** — 심어둔 관습을 단 한 건도 감점하지 않음. cliche_count(참고 카운트)는
  4~7로 여전히 세면서 이슈화는 0 — P5 정합이 광고 밖 장르로 일반화됨을 확인.
- CRITICAL 3/10 (전부 causality — 클리셰 무관):
  - fantasy: Fear→Determination 급변에 매개 부재 — **정당** (배터리 B의 mut2형 패턴을 실전에서 포착).
  - comedy: "봄이가 자기 생일 케이크를 직접 준비하는 구조 모순" — **정당** (프리셋 스토리 자체의 균열을
    정확히 지적 — 입력 전제의 결함 검출 사례).
  - romance: "편지 읽는 행위가 씬 전환 사이에 생략" — **오탐 (사실 오인)**. 스팟 체크 결과 scene_3
    액션에 "카페 안에서 편지를 읽던 도현이…"가 명시돼 있음. 인용 검증 없이 CRITICAL을 낸 케이스 1/10.
- **default-on 관찰**: 첫 패스 scenes의 3/10이 passed=false → 그대로 켜면 ~30% 재생성률(오탐 1 포함).
  default-on 전 선행 조건: (a) CRITICAL의 사실 인용을 코드로 대조(이슈가 지목한 텍스트 실재 확인) 또는
  (b) 재생성 대신 배지 표면화(독트린 §0.5 가정 라벨 패턴). — 후속 판단 항목.

## 5. 부수 발견 (실측)

- **[실측] C축 키 실배선은 `CLAUDE_API_KEY`** (`llm/claude.ts`) — ANTHROPIC_API_KEY 아님. 하네스 주석 교정됨.
- **[실측] 장편(D7) scenes는 C1 실행 불가** — max_tokens=4096 초과로 응답 잘림(2회 재현). C1을 살려도
  장편에선 maxTokens 상향 또는 청킹 필요. dispatch가 maxTokens를 전달하는지 확인부터 — 후속 항목.
- **[관찰] 스텁 personality 교란은 실측상 미미** — 구판 오탐의 근거는 전부 클리셰·장르 계약 오해였고
  personality 부재를 이유로 한 오탐은 관찰되지 않음.
