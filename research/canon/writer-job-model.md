## 0. 정식화 — writer는 "결정 압축기"다

유저 입력은 **불완전한 결정 집합**이고, 영상 생성기는 **완전한 결정 집합**(샷마다 첫 프레임·모션·에셋·초)을 요구한다. writer의 일은 그 사이를 채우는 것 — 창작이 아니라 **결정 완성(decision completion)**이다.

**unknown의 정체 (오해 방지)**: knowns는 유한하다(유저가 실제로 말한 것 + 논리 유도분) — 원장에 기록되는 건 이것과 "물어볼 소수 후보"뿐이다. unknowns는 무한하고 **영원히 목록화되지 않는다** — 생성 행위 자체가 prior에서 샘플링하며 암묵적으로 채운다. unknown은 리스트가 아니라 **분포**다.

따라서 모든 빈칸엔 채움의 우선순위가 있다 (개정: 3→5단계):

1. **유저가 정한 것** → 그대로 (변경·재해석 금지)
2. **유저가 정한 것에서 논리적으로 따라오는 것** → 유도 (발명 아님)
3. **뿌리급 × prior 다봉 × 단서 없음** → **예산제 질문** (§0.5 — 유일하게 유저에게 되묻는 예외)
4. **뿌리~가지급인데 질문 예산에서 밀린 것** → **가정 라벨** (prior로 채우되 가정을 보이게 — "가정: 열린 결말(관습 디폴트)" 배지)
5. **나머지 무한 파편 (잎)** → **조용한 관습 디폴트** (그 장르에서 관객이 기대하는 가장 전통적인 선택)

5번이 이 제품의 승부처다. "모호하면 묻는다"가 아니라 "모호하면 관습대로"가 계약이므로, **클리셰는 버그가 아니라 명세다.** 놀라움(발명)은 유저가 입력에 심어둔 차별점을 증폭할 때만 정당하다.

## 0.5 묻기 규율 (ask discipline) — "묻기는 후회 관리, 채우기는 prior"

질문의 목적은 **정보 수집이 아니다** — prior는 관습 정보를 이미 충분히 안다. 전문가에게 물어서 얻는 것은 **갈림길의 귀속**: prior가 여럿으로 갈라지는 지점에서 그 선택이 *유저의* 선택이 되게 하는 것. 즉 넛지는 후회 관리다.

- **레이어 두 겹**: 정적 ask = producer 게이트(장르·캐스트·배경·앵커 — 이미 구현됨). 동적 넛지 = 입력을 읽고 *이 스토리 고유의* 고레버리지 모호점을 찌르는 것 (신규 제안 영역).
- **예산제 3조건** — 셋 다 동시 성립할 때만 묻는다:
  1. **뿌리급 파급** (하류 전체가 상속 — 모델전략 문서의 파급 축 재사용)
  2. **prior 다봉(多峰)** (관습 후보가 여럿이고 서로 멀 때 — 관습이 하나로 수렴하면 뿌리여도 안 묻는다)
  3. **입력에 단서 없음** (우선순위 2번 논리 유도로 못 푸는 경우만)
  질문 0개인 프로젝트가 **정상이고 다수**여야 한다. "최대한 knowns로 전환"은 목표가 아니다.
- **질문의 형태 — 빈칸 금지, 갈림길 제시**: "이대로면 A로 간다(관습 디폴트). B로 비틀까?" — 디폴트 선선택, 1클릭 확인 or 무시. 무시 = 디폴트 채택 = 원계약("먼저 묻지 않는다")으로 자연 강등이라 질문이 흐름을 절대 막지 않는다. 전문가는 명세보다 교정에 강하다(recognition > recall) — 추상 빈칸은 심문이고, 구체적 갈림길 제시가 잠재된 선호를 깨운다. 어려움은 질문 수와 빈칸 크기에서 오므로 이 형태면 전문가·비전문가 모두 안전.
- **가정 라벨(우선순위 4)이 질문보다 먼저다**: 전문가는 결과물을 보면 어디가 틀렸는지 아는 사람들 — 사전 질문보다 초안+가정 라벨이 사고를 더 잘 깨울 수 있다 (아키텍처 "stale은 정보, 행동은 명시적"의 UX 버전). 사전 질문은 **재생성 후회가 비쌀 때만** (장편·뿌리 결정을 잘못 뽑으면 비싸다).
- **롤아웃 전략 (데이터 기반 승격)**: 가정 라벨을 먼저 깔고 → 유저가 자주 뒤집는 가정(뒤집힘률 높은 것)을 사전 질문으로 승격한다. 질문 예산의 근거를 추측이 아니라 라벨 수정률 실측에서 얻는다.

## 1. writer가 봉사하는 세 가지 진실

모든 스테이지 프롬프트는 이 셋 중 무엇을 다루는지 자각해야 한다. 섞이면 (S1 사다리처럼) 어휘 충돌이 난다.

| 진실 | 성격 | 예 | 위반 시 |
|---|---|---|---|
| **원장 (ledger)** | 유저·상류 확정값 = 불변 | styleAnchor, cast slug, runtime, format | 유저 배신 (앵커 무시 = d6208bba 실측 사고) |
| **물리 (physics)** | 생성 백엔드의 법칙 = 협상 불가 | 클립 2~8초, 클립당 동사 1~2, 카메라+액션 동시 금지, 에셋 ID 실재 | 생성 실패·"뇌 아픈 영상" |
| **관습 (prior)** | 필름 문법·장르 기대 = 디폴트, 근거 있으면 이탈 가능 | 씬 문법, 샷 사이즈 흐름, 180°축, 장르 클리셰 | 어색하지만 치명적이진 않음 |

판별 질문: **"이 제약을 어기면 무엇이 깨지나?"** — 유저 신뢰(원장) / 영상 자체(물리) / 세련됨(관습). 프롬프트에서 세 층은 다른 강도로 말해야 한다: 원장="그대로 써라", 물리="절대 못 넘는다", 관습="이유 없으면 따르라".

## 2. 직무 6분해 — 서사를 영상으로 완성하려면 반드시 참이어야 하는 것

파이프라인이 몇 스테이지든, 아래 여섯 job이 어디선가 수행돼야 한다. (현 스테이지 매핑 병기)

1. **원장 정리** — known 고정 + 물어볼 후보 선별(§0.5 3조건). unknowns 자체는 목록화 대상이 아니다 — 생성이 곧 prior 샘플링(§0). *(현재: 전담 없음 — styleAnchor(v0)·slug 규칙(s3)에 국소 구현. 동적 넛지·가정 라벨은 미구현)*
2. **서사 완성** — 형태(구조 유형) 선택 + 규모(비트·씬) 배분 + 인물 아크 + CDQ. *(S1+S3)*
3. **시간 예산** — runtime → 씬 초 → 샷 초. 산수는 코드가, 배분 판단은 모델이. *(S3 "총합≈N초" 한 줄에 위임 — §감사 참조, 최약 지점)*
4. **세계 상수 확정** — 스타일·팔레트·인물·장소의 전역 불변량. *(V0+V1+V2)*
5. **연출 번역** — 비트→샷, 물리 제약 안에서 리듬 저작. *(V3+decoupage+V4)*
6. **계약 검증·조립** — 결정론 우선, LLM은 의미 스팟체크만. *(C1+C2+V5 — C2가 이미 이 방향으로 진화함)*

주의: job 경계 ≠ 스테이지 경계일 필요 없다. 스테이지 경계가 정당한 건 (a) 유형이 갈릴 때(저작→수행), (b) 모델 배치가 갈릴 때, (c) 체크포인트/부분 재실행 단위일 때뿐. 이 셋에 해당 없는 경계는 정보 손실·모순 기회·레이턴시만 만든다.

## 3. 프롬프트 작성 독트린 (P1~P6)

감사에서 발견된 결함들의 일반화. 재작성 시 이 여섯을 체크리스트로.

- **P1. 원장 3블록** — 모든 저작 스테이지 프롬프트는 같은 서두 구조를 갖는다:
  `[불변 입력 — 변경·재해석 금지]` / `[네가 채울 빈칸 — 이 스테이지의 저작 범위]` / `[채움 원칙 — 모호하면 그 장르의 관습 디폴트. 유저 입력의 차별점만 관습보다 우선]`.
  → 본질 문장을 개별 규칙의 우연한 합이 아니라 **프롬프트 아키텍처**로 만든다.
- **P2. 물리 상수는 단일 소스** — 샷 초 대역, 액션 예산, 프롬프트 자수 같은 백엔드 법칙은 코드 상수 한 곳에서 모든 프롬프트에 주입한다. 산문에 박아 넣는 순간 스테이지마다 어긋난다 (실증: 2~8s vs 5~15s vs 50~80자 vs 50~100자 — 감사 §전역-1).
- **P3. 축 어휘 분리** — 형태/규모/물리를 한 리스트에 섞지 않는다 (S1 사다리 분석 `../records/2026-07-21-structure-form-vs-scale-axes.md`의 일반화).
- **P4. 프롬프트는 실제 배선만 말한다** — default-off 스테이지, 대체된 명칭(S0/S2), 죽은 모드(Compact), 코드가 버리는 출력 필드를 전제로 한 서술 금지. 아키텍처 룰 "같은 진실 이중 기록 금지"의 프롬프트 버전 — 코드가 진실이고 프롬프트는 그 파생이다.
- **P5. 클리셰 정책 정합** — 검수(C축)는 "관습적 선택"을 감점하지 않는다. 감점 대상은 (a) 인과·핍진성 붕괴, (b) 유저가 준 차별점의 실종뿐. (§0의 3번 원칙과 C1 프롬프트가 현재 정면 충돌 — 감사 참조)
- **P6. 산수는 모델에게 시키지 않는다** — "총합 ≈ N초" 같은 제약 충족을 모델 재량에 두지 않는다. 코드가 예산표(씬 수 × 초 범위)를 계산해 주고, 모델은 배분의 *판단*만 한다. 검증도 코드가 한다.
- **P7. 묻기는 예산제 예외, 형태는 갈림길** — 유저에게 되묻는 것은 §0.5 3조건(뿌리×다봉×단서없음)을 전부 통과할 때만. 질문은 빈칸이 아니라 디폴트 선선택된 갈림길("이대로면 A로 간다 — B로 비틀까?")이어야 하고, 무시가 항상 안전해야 한다. 질문 예산에서 밀린 뿌리~가지급 가정은 라벨로 표면화한다.

## 4. 이 독트린 자체의 검증 (메모)

독트린도 가설이다. 각 P의 효과는 감사 문서의 실험 백로그(§5)에 개별 매핑돼 있다 — 특히:
- P1 효과 = 실험 E1 (원장 3블록 유/무 A/B: 유저 확정값 이탈률·발명 빈도)
  → **판정 (2026-07-21): 미채택 — 바닥 효과.** 16쌍(2+10장르 프리셋)·32 run 전부 원장 위반 0 동률.
  P1은 이미 국소 규칙(오픈 캐스트/로케이션 조항)으로 구현돼 있다 — P1을 "서두 형식 요구"가 아니라
  **프롬프트 재작성 시 보존 목록**으로 재해석한다 (E9·E13에서 국소 규칙 유실 금지). 재프로브: 모델 교체
  시 / 운영 DB 원장 위반 실측 시. (`../experiments/foundation/2026-07-21_ledger-blocks/result.md`)
- P2 효과 = 실험 E7 (대역 통일 후 샷 duration 분포 회귀)
- P5 효과 = 실험 E5 (C1 오탐×검출력 직접 측정)
  → **판정 (2026-07-21): 채택 — C1 재정의 반영.** 구판은 정상 산출물 4/4 클리셰 감점 + 장르 관습(제품
  매직 모먼트)을 DEM CRITICAL로 오판. 재정의판(클리셰 감점 제거·장르 계약 조항·differentiator 신설)은
  오탐 0 + 주입 결함 검출 4/4 유지 + 10장르 일반화 확인. (`../experiments/validators/2026-07-21_storycheck-redefine/result.md`)
- P6 효과 = 실험 E3 (예산표 주입 유/무: total_estimated_seconds의 runtime 수렴)
  → **판정 (2026-07-21): 채택** — E3b 2층 예산 프로덕션 반영. (`../experiments/time-budget/2026-07-21_budget-injection/result.md`)
- P7 효과 = 실험 E14 (3조건 판별식의 실용성 오프라인 검증 → 가정 라벨 뒤집힘률 실측 → 질문 승격)
