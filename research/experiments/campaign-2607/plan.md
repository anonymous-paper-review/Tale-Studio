## 1. 한 장 요약 (보고용)

**목적**: writer의 12개 LLM 프롬프트를 재작성한다. 단, 감으로 고치지 않는다 — 모든 변경을 "진단(문제가 진짜인가) → 처방(고치면 나아지는가)" 실측으로 게이트한다.

**핵심 판단 — 변경은 두 종류다**:
- **청소(W그룹)**: 죽은 문구·자기모순·낡은 명칭 제거. 실험이 필요 없다(옳음이 코드 리딩으로 증명됨). 회귀 확인만 하고 일괄 적용.
- **개선(E그룹)**: "이렇게 바꾸면 더 나아질 것"이라는 가설. 각각 A/B 실측으로 판정한다.

**로드맵 4단계**:

| 단계 | 내용 | 산출 | 예상 규모 |
|---|---|---|---|
| **P1 진단** | 지뢰 실증 + 베이스라인 확보 (프롬프트 변경 없음) | 문제의 실측 증거, 개선 판정용 기준값 | run 수십 회 + DB 감사 |
| **P2 청소** | W1~W5 일괄 적용 + 회귀 배터리 R1 | 퇴적층 없는 프롬프트 (재작성의 깨끗한 출발점) | 코드 반나절 + 회귀 run |
| **P3 개선** | E그룹 개별 A/B (사다리·예산표·원장 블록·존폐 결정들) | 채택/기각이 판정된 프롬프트 개선 | 실험 8건 |
| **P4 구조** | 스테이지 병합 실험 (12콜→7콜 후보) | 파이프라인 구조 결정 | 실험 2건 |

**순서의 이유**: P1이 없으면 P3의 "나아졌다"를 판정할 기준값이 없다. P2가 없으면 P3의 A/B가 퇴적층 노이즈와 섞인다. P4는 프롬프트가 안정된 뒤에만 의미 있다.

## 2. W그룹 — 실험 불요 청소 (P2에서 일괄, 회귀 R1로 게이트)

근거는 전부 감사 문서의 `[코드]` 확정 발견. 각각 독립 커밋으로.

| # | 변경 | 근거(감사) |
|---|---|---|
| W1 | **물리 상수 단일 소스화** — 샷 초 대역(2~8s, 예외 10)·프롬프트 자수(모션 50~80자)를 코드 상수로 만들어 decoupage/V4/V5 프롬프트에 주입. V4의 "5~15초" 잔재 삭제 | 전역-1, 1.10-① |
| W2 | **V4 죽은 지시 제거** — "story_beat_ref 1:1 매핑"(코드가 덮어씀), 데쿠파주 모드의 "shot_count_target ±1"(모순 동거) | 1.10-②③ |
| W3 | **죽은 명칭·서사 정리** — S3/C1의 "S0~S3" 명칭, V0의 "Mid Preview 제안을 채택"(항상 빈 seed 현실 반영) | 1.2-②, 1.5-② |
| W4 | **genre.format → v0 코드 고정** — 유저 확정 화면비를 LLM 경유 없이 복사 (원장 값은 모델을 거치지 않는다) | 1.5-① (E2 흡수 — 이탈률 측정 불요, 설계 원칙으로 결정) |
| W5 | **V5 fallback 자수 일치** — 50~100자 → V4와 동일 대역 (언어는 E11 판정 전 현행 유지) | 1.12-① |

**R1 회귀 배터리** (W 적용 전/후 각 1회): 4 프리셋(shorts/ad/kishoten/loop — feature는 P1의 E3a에서 별도) × 지표 M1~M6 비교. 판정: 악화 지표 0개면 통과. 특히 W1 후 duration 분포(M3)에서 >8s 샷 비율이 줄었는지 관찰(감사 E7의 흡수).

## 3. E그룹 — 실험 카드

각 카드: **가설 → 방법 → 지표 → 판정(사전 확정) → 채택 시 변경되는 것**. 지표 정의는 §5.

### Phase 1 — 진단

**E0a. S1 깊이 사다리 프로브** ⬜
- 가설: 사다리의 형태 어휘(D2 "setup→action→result", D3 "3-act 또는 기승전결")가 저깊이×비선형 콘텐츠에서 형태 선택을 꺾는다 (축 분석 문서 §7).
- 방법: 프리셋 2개 추가 — `kishoten-d2`(30s 광고·갈등 없는 대비), `loop-d1`(한 순간 순환). narrativeStructure 각 3 run.
- 지표: M6 (structure_type이 콘텐츠를 따르는가).
- 판정: 3 run 중 2+ 회 콘텐츠 형태를 놓치면 "꺾임 확정" → E0b 근거. 안 꺾이면 사다리 재작성 우선순위 하향(잠복 결함 기록만).
- 변경 대상: (진단이므로 없음)

**E3a. 시간 예산 붕괴 실측** ⬜
- 가설: runtime 정합을 아무도 소유하지 않아(감사 전역-2) 장편에서 총합 미달 또는 씬 초 뻥튀기가 난다. 특히 "씬당 1~3액션" vs "씬당 200초"(feature)는 산술 모순.
- 방법: shorts/ad/feature × scenes 각 3 run (세션 열린질문 #1과 동일 run에서 관찰 확장).
- 지표: M1(runtime 편차율), M2(씬 내부 정합).
- 판정: feature에서 M1 > 20% 또는 M2 괴리 중앙값 > 3배면 "붕괴 확정" → E3b 착수.
- 변경 대상: (진단)

**D1. DB 감사 묶음** ⬜ (LLM 비용 0 — 기존 run 데이터만)
- 내용: ① structure_type 분포 재감사(앵커 수정 효과 — 세션 열린질문 #3) ② 씬 purpose 히스토그램(감사 E4) ③ decoupage uncovered_beats 분포(E10) ④ C2 hook_type 채움률(E12a) ⑤ V5 extraction_summary 집계(fallback 빈도).
- 판정: ②가 conflict/exposition 편중이면 P3에 "장르 관습 씬 흐름 힌트" 실험 추가. ③에서 REMOVE 남용 보이면 validator 추가. ④가 희박하면 E12b의 제거 근거 강화.
- 변경 대상: (진단 — P3 실험 목록을 확정하는 입력)

### Phase 3 — 개선 (각각 독립 A/B)

> **대기열 재정렬 (2026-07-21, 제품 오너 지시 "핵심 검증부터")**: Phase 1~2와 E3b가 검증한 것은
> 독트린의 위생 계층(P2·P4·P6)뿐이다. 제품 본질 문장과 직결되는 **P1(원장 보존)·P5(클리셰는 명세)는
> 미검증** — 이 둘을 검증하는 E1·E5를 최우선으로 올리고 나머지(E12b·E6·E8·E11)는 후순위로 내린다.
> E5는 방법을 상관설계(10 full run)에서 직접 측정 2배터리(오탐·검출력)로 개정 — 개정 근거는 카드에.

**E0b. S1 깊이 사다리 재작성** ⬜ (선행: E0a 꺾임 확정)
- 가설: 사다리를 규모 언어(비트 수·서브플롯·층위)로만 재작성 + 형태×규모 결합 제약 명시(축 분석 §6)하면, 꺾임이 사라지고 기존 4셀 회귀 없음.
- 방법: 재작성판으로 E0a 2셀 + 기존 4셀(shorts/feature/kishoten/loop) 재실행.
- 지표: M6. 판정: 신규 2셀 교정 + 기존 4셀 유지 = 채택.
- 변경 대상: `s1_structure.ts` 시스템프롬프트 D1~D7 블록 전면 교체.

**E3b. 씬 시간 예산표 주입 — 2층 예산** ⬜ (선행: E3a 붕괴 확정 ✅ + 제품 결정 ✅)
- **제품 결정 (2026-07-21, 제품 오너)**: 장편(D6~D7)은 MVP 겉보기 스코프 — "30~60분도 되는 척"이 목표이지 정직한 커버(~1,100샷)가 아니다. → 처방을 2층으로 확정:
  - **정직 대역 (D1~D5)**: 씬 초 = 액션 수 × 5~8s(+소폭 여유)로 실제 결합. 코드가 예산표 계산·주입 + scenes 직후 검증(책임 스테이지 확정).
  - **표면 대역 (D6~D7)**: total ≈ runtime 유지(E3a 실측상 이미 잘 됨), 씬당 액션 완만 비례(3~6) + **전역 샷 총량 정책 상한**(대표 샷 스토리보드) → decoupage에 샷 예산 힌트로 전달. M2 초과는 의도된 것 — 산출 메타에 표시(coverage_mode: 'representative' 류).
  - 상한은 물리 상수(physics.ts)가 아니라 **제품 정책 상수** — 별도 구분 주석. 장편이 진짜 스코프가 되면 이 상수만 올린다.
- 가설(P6): 암산 위임 제거로 정직 대역 M2가 수렴하고, 표면 대역은 폭주 없이 겉보기 정합을 유지한다.
- 방법: 주입판 vs 현행 × shorts/ad/kishoten(정직 대역)·feature(표면 대역) 각 3 run.
- 지표·판정: 정직 대역 M1 ≤ 10% + M2 중앙값 ≤ 1.5 / feature M1 ≤ 10% 유지 + 씬·샷 수 상한 내 + 에러 0 = 채택.
- 변경 대상: `s3_scenes.ts`(sceneCountHint 산문 → 코드 예산표 + 검증), `decoupage.ts`(샷 예산 힌트 수용), 정책 상수 모듈.

**E1. 원장 3블록 서두** ✅ 미채택 (방법 개정 2026-07-21 · 재정렬로 최우선)
- 가설(P1): [불변 입력]/[저작 범위]/[채움 원칙: 관습 디폴트] 표준 서두를 달면 유저 확정값 이탈과 무근거 발명이 준다.
- 방법(개정): S3·V4 적용판(`WRITER_LEDGER_BLOCKS=1` env 게이트) vs 현행 × **ad·ledger**(원장 밀도
  프리셋 신설 — 고정 소품·고정 결말·slug≠표시명 캐스트/로케이션) 각 3 run, 체인 S1→S3→decoupage→V4.
  4→2 프리셋 축소 근거: 길이 축은 E3에서 소진, E1의 축은 원장 압력 — 원장이 얇은 프리셋은 신호가 없다.
- 지표(M4 확장): ① 로케이션 id 위반 — **raw 응답 기준**(정규화 전, llm_calls에서 회수) ② new_characters
  남발(스토리 외 인물) ③ cast drop(원장 인물 미등장) ④ V4 미지 character_id 참조 ⑤ 차별점 토큰 생존
  (ledger 프리셋: 노란 우산 / 마지막 컷 지정 / "비 오는 날만 영업" 역전 설정). 부수 감시: M1·M5 회귀.
- 판정(사전 확정): B의 원장 위반 합계(①~④) < A **그리고** ⑤ 생존율 ≥ A **그리고** M1/M5 악화 없음 = 채택
  → 전 저작 스테이지로 확산. 위반 합계 동률·⑤ 동률이면 미채택(서두 표준화는 문체 통일 가치만 기록).
- 변경 대상: 전 저작 스테이지 시스템프롬프트의 서두 구조 표준화.

**E5. C1 storyCheck 존폐** ✅ 재정의판 채택 (방법 개정 2026-07-21 — 상관설계→직접 측정 · 재정렬로 최우선)
- 가설: ⓐ C1은 관습적(의도된 클리셰) 산출물을 감점한다 — P5 정면충돌. ⓑ 정작 진짜 결함(인과 붕괴·핍진성
  붕괴·차별점 실종)을 잡는지는 불명.
- 방법(개정): **배터리 A(오탐)** — 정상 판정된 기존 E3b 산출물 5건(shorts×2·ad×2·feature×1)에 C1 실행 →
  클리셰 카테고리 WARNING+ 빈도. **배터리 B(검출력)** — ad 산출물에 결함 주입 변이 2종(①데우스 엑스
  마키나+제품 차별점 실종 ②핍진성 붕괴+감정 불연속) × 각 2 run → 주입 결함을 CRITICAL로 특정하는가.
  개정 근거: 10 full-run 상관설계는 간접·고비용 — 존폐 판단 축은 오탐×검출력 2개로 직접 측정 가능.
  주의(교란 통제): 하네스 캐스트는 personality가 빈 스텁 — 핍진성 카테고리 절대치는 오염 가능. 판정은
  같은 프리셋의 (정상 vs 변이) **차분**과 클리셰 카테고리(스텁 무관)만 쓴다.
- 판정(사전 확정): 검출력 없음(주입 4 run 중 2+ 미검출) = **삭제**. 검출력 있음 + 오탐 있음(정상 5건 중
  2+에 클리셰 WARNING+) = **P5-정합 재정의**(클리셰 감점 제거, 인과 붕괴·차별점 실종만) 후 재측정.
  검출력 있음 + 오탐 없음 = 프롬프트 유지, default-on 여부만 후속 판단.
- 변경 대상: `c_validation_1.ts` 삭제 또는 프롬프트 재정의 + `resolveSkip` 기본값.

**E6. midPreview 존폐** ✅ 판정(07-22 저녁): 삭제 채택 (E6b 경유 — 코드 제거 집행)
- 가설: 항상 skip인 현실에서 켜도 V축 산출 개선 없음 — 삭제가 맞다.
- 방법(구체화): on/off × **ad·horror-mansion** × 2 run — 1차는 seed의 최직접 소비자 **V0 산출만** 비교
  (V0~V4 전체 rubric은 고비용; V0에서 차이가 없으면 하류 차이도 seed 경유로는 없음). OFF팔은 실
  파이프라인 skip 경로와 동일한 emptyMidPreview() 주입.
- 판정: V0 산출(medium·style·render·texture) 차이 무의미 + midPreview 비용 실측 = 삭제 권고. 유의미 = 소비처 배선부터 재설계.
- 변경 대상: `mid_preview.ts` + steps.ts 배선 + V0~V4의 seed 참조 문구.

**E8. v1 actVisualArc 배선 복원 vs 제거** 🟠 판정(07-22): 기준 수립 후 재측정 → E8b (방법 구체화 2026-07-21)
- 가설: v1의 시간축 페이로드는 착지가 없어(V3 미입력) 현행에선 무효 — V3에 주입하면 막별 시각 진행이 생기고, 그게 안 예쁘면 제거가 맞다.
- 방법(구체화): 2안 실측 — ctl(현행 무배선) vs arc(주입, `runSceneCinematography` 선택 파라미터 +
  `WRITER_V3_ARC=1`) × **kishoten·family-jjigae**(4막 기승전결 — 막별 신호에 충분) × 2 run.

- 남는 선택지: 품질 축 변형 E12c-B — "렌더 문장 전담 재작성 패스 vs 현행 직산출" 블라인드 비교(약 100~130콜, 채택 시 상시 호출 증가 대가). 실행 여부는 오너 판정.

**E13b. S1+S3 병합 정식 구현 재검증** 🟡 측정 완료 — 판정 대기 (2026-07-22, 사전 기준 4/4 통과 — 채택 상신)
- 판정 배경: 조건부 진행 — E13의 선행 3건을 상세 수행 후 재검증.
- 내용: ① 정식 병합 스테이지 구현(env 게이트, 기본 off — S3 후처리(장소 정규화·병합·예산 검증·커버리지) 전부 이관) ② 체크포인트 재설계(병합 1콜 산출을 기존 구조/장면 저장 슬롯 양쪽에 기록해 하류·재실행 단위 불변) ③ 재검증 배터리(4소재 × 2 + ad 추가 3회로 구조 분산 표본 확대).
- 판정(사전): 막 커버리지 누락 0 + 장소 표기 오염 0(후처리 이관 증명) + 구조 프로브 정답 + 시간 절감 유지 = 채택 상신.
