# 전편 카피 실험 설계 — "배운 것 전부 묶으면 현행 백엔드를 이기는가" (BKM vs BASE)

> **한 줄**: 원본 전편(67초·30컷)을 같은 시나리오에서 두 파이프라인으로 재현한다 —
> **BASE**(실제 writer 백엔드 그대로) vs **BKM**(입력 포맷 실험에서 배운 것 전부 묶음) —
> 그리고 R(원본 프레임 상한)과 나란히 오너 블라인드 판정. 요소 격리는 하지 않는다:
> **묶음이 이기면 통째 배선, 원인 규명은 사후 뺄셈 실험**으로.
>
> 설계일 2026-07-23 · 선행: [입력 포맷 실험 판정](../2026-07-23_input-format/result.md) §6 · 상태: 오너 확정 대기

## 1. 왜 이 구조인가

입력 포맷 실험의 판정이 원인 4개를 확정했다: ① 카메라 연출층 부재 ② 사건(스토리) 층 부재
③ 이미지 QC 게이트 부재 ④ A팔이 실백엔드가 아니었음. 원인별 팔을 파면 조합 폭발이므로,
이번엔 **시스템 수준 질문 하나**로 묶는다: "전부 고치면 제품 현행보다 확실히 나은가?"
- 이기면 → 묶음째 제품 배선 상신. 기여 요소가 궁금하면 승리 묶음에서 하나씩 빼는 후속 실험.
- 져도 → "아는 걸 다 넣어도 안 됨" = 병목이 모델 계층이라는 강한 신호 (H2와 교차 확인).

## 2. 가설 (사전 고정)

- **H1 (주)**: BKM은 BASE보다 원본 전편 카피에서 오너 블라인드 순위 명확 우위다.
- **H2 (부)**: 전 구간에서도 R의 한계(임의 카메라 무브·샷 연결 미달)가 재현된다.
  재현되면 R↔원본 격차는 입력·프롬프트로 못 넘는 모델 한계로 확정 → 모델/기능(멀티샷) 검토 근거.

## 3. 팔 구성

| 팔 | 파이프라인 | 영상 모델 |
|---|---|---|
| **BASE** | 시나리오(줄거리 상세·**카메라 지시 없음**) + 장르 태그(스릴러) → **실제 writer 백엔드 그대로**. **카메라 무브를 포함한 연출 일체를 writer가 자생 생성** — 수기 개입 금지 (오너 지시 2026-07-23) | Seedance 2.0 (시작 프레임만 — 백엔드 산출 형태 그대로) |
| **BKM** | 같은 시나리오 → B1 골격(시작+끝 프레임) + 아래 §4 묶음. **카메라 4요소는 원본 분석(콘티·레퍼런스 해석)을 근거로 수기 상세 작성 — 고점 측정** | Seedance 2.0 |
| **R** | 원본 30컷 시작·끝 프레임 직입력 (상한 눈금) | Seedance 2.0 |
| β-프로브 | Seedance **멀티샷 모드**(`multi_shots` 파라미터 실측 존재)에 사건 맥락+컷 지시, 15초 청크 1~2콜 | 판정 제외 — "모델이 컷편집을 네이티브로 하는가" 정찰 |

**모델 통일 (오너 확정 2026-07-23)**: 전 팔 Seedance 2.0 — BASE도 포함. 팔 간 차이는 오직
"프롬프트·이미지를 누가/어떻게 만들었나"뿐이다 (이전 판의 happy_horse 교란 제거).
BASE의 영상 입력 형태는 백엔드 산출물 그대로(시작 프레임+텍스트, 끝 프레임 없음) — 그것까지가
"현행 제품"의 일부다.
**레인**: 전 팔 힉스필드 — fal×Seedance는 포토리얼 얼굴을 파트너 검증으로 차단함이 8케이스
격자 프로브로 확정됨 (프롬프트 무관·이미지 원인, 실존/생성 무구분, 애니·무인물·극소 인물은 통과.
과거 fal 실존인물 영상 성공은 전부 happy_horse였음 — `../2026-07-23_input-format/assets/probe-fal/`).
**비교의 의미**: BASE는 "장르+줄거리만 주면 제품이 연출을 얼마나 스스로 뽑는가"(자생 연출력),
BKM은 "원본을 분석한 사람이 쓸 수 있는 최고 연출"(고점). 두 팔의 격차가 곧 writer 연출 계층의
개선 여지 크기다.

## 4. BKM 묶음 정의 (입력 포맷 실험 판정 §6의 처방 전부)

1. **카메라 연출층**: 샷마다 카메라 4요소(렌즈·앵글/구도·무브·광학) **긍정형** 명시.
   "지시 외 무브 금지"류 네거티브는 긍정 지시가 있어야 작동한다 (R s2 기각 사례).
2. **사건 맥락층**: 샷마다 "왜 이 행동을 하는가" 1~2문장 (스릴러 비트 — 세면대 소리, 놀람 등
   **오디오 이벤트 포함**). 콘티가 구도만 옮기고 사건을 버렸던 것이 무맥락 생성의 원인이었다.
3. **이미지 QC 게이트**: 시작·끝 프레임은 신원(정본 대조)·시선 방향·소품 접촉(완드-입술 등)·
   카메라 위치(원본 프레임 대조) 4항목 눈 검수 통과분만 I2V 투입. 불합격은 재생성.
4. **촬영/편집 분리 (테이크 프레임 — 오너 교정 2026-07-23)**: "생성 클립 1개 = 완성본 샷 1개"
   가정을 폐기한다. **생성 = 촬영(테이크), 조립 = 편집.**
   - 테이크는 전부 4~8초로 생성 — Seedance 최소 4초는 "카메라가 최소 4초 돈다"일 뿐, 촬영은
     원래 편집보다 길게 찍으므로 제약이 아니다. 완성본의 1초 컷은 테이크에서 오려 쓴다.
   - **테이크 재사용**: 반복 구도(마스터 후렴 4~5회)·같은 사건 클러스터의 앵글은 테이크 1개에서
     여러 컷을 오려 쓴다 → 컷 30개 ≠ 생성 30개. 테이크 계획은 conti_full.md에서 확정
     (예상 테이크 수 30 미만 — 비용·일관성 동시 이득).
   - 산출 구조: **테이크 계획**(구도별 무엇을 몇 초 찍나) + **편집 계획**(어느 테이크의 어느
     구간을 어느 컷 자리에 넣나) 2계층. 끝 프레임은 테이크의 도착점(방향타)으로 유지.
   - (구 "컷 재절단 규칙" 논의는 생성=샷 가정에서 생긴 가짜 결정이었으므로 폐기.)
5. 신원 정본·연속성 바이블·네거티브 배터리는 기존 유지 (검증된 층).

## 5. 공통 준비물 (선행 작업 — 생성 전 오너 확인)

- **가. 전편 콘티화**: 원본 67초 → 30컷 시작/끝 프레임 추출(컷 시각은 레퍼런스 해석 문서 부록
  재사용) + **오디오 트랙 분석**으로 사건 비트 기술. 산출: `conti_full.md`.
- **나. 시나리오 역변환**: 원본을 우리 제품 입력 형식(스크립트/시나리오)으로 옮긴다 — BASE와
  BKM이 **같은 시나리오**에서 출발해야 비교가 성립. **줄거리·사건·분위기는 디테일하게 써도 되나
  (오너 허용), 카메라·앵글·편집 지시는 한 글자도 넣지 않는다** — 시나리오에 연출이 새면 BASE의
  "자생 연출" 측정이 오염된다. 장르는 태그(스릴러)로만 전달. 초안 작성 후 오너 확인 1회.

## 6. 평가 (사전 고정)

**정성 (판정의 본체)**: 오너 블라인드 — 원본·BASE·BKM(·R) 나란히, 구간별(도입/립 구간/사건
전환/결말) 순위 + 전체 순위. 라벨 없는 모자이크 + 키 파일 방식 재사용.

**정량 (보조)**:
1. 카메라 위치 정합 — 샷별 클립 중간 프레임 vs 원본 해당 프레임 구도 대조 (신규, 오너 1순위)
2. 첫/끝 프레임 SSIM (기존 지표 연속성)
3. 신원 일관성 — 샷 간 인물 드리프트 카운트
4. 컷 경계 보존 — 조립본에서 의도한 30컷이 유지되는가

**판정 규칙**: H1은 오너 구간 순위에서 BKM이 BASE를 과반 구간에서 이기면 채택 →
BKM 묶음 제품 배선 상신. 못 이기면 묶음 해체 + 병목 재분해. β 결과는 기록만.

## 7. 비용·순서

| 단계 | 규모 |
|---|---|
| 선행 가·나 (콘티화+시나리오) | 생성 0 (완료: conti_full.md / 시나리오는 스토리 확인 대기) |
| BKM 이미지 (테이크 시작·끝 + QC 재생성) | 편집 ~60콜 |
| 클립: BKM 24테이크(~108s) + R 29컷(~125s) + BASE(~100s 추정) + β 2 | **~1,925크레딧 (1,700~2,200)** — 테이크 기준 재산정, conti_full.md §3 |
| 조립·대조·블라인드 | ffmpeg (기존 도구 확장) |

순서: 가·나 → 오너 확인 → R(눈금 먼저) → BKM·BASE 병렬 → β → 조립 → 블라인드 판정.

## 8. 오너 확정 이력 (전 항목 해소 — 2026-07-23)

**확정됨 (2026-07-23 오너 지시)**:
- 대상 = 원본 전편(처음~끝) ✓
- BASE = 실제 writer 백엔드, 카메라 무브는 writer 자생 생성(수기 금지), 입력은 장르 태그+상세 줄거리 ✓
- BKM = 원본 분석 기반 카메라 무브 수기 상세 작성(고점 측정) ✓
- 방향 = 컷편집 허용 + 카메라 정보 + 스토리 요소 프롬프트 주입 ✓
- 전 팔 영상 모델 Seedance 2.0 통일 — BASE 포함 (happy_horse 교란 제거, 레인은 힉스필드 고정) ✓
- R 상한 = 전범위(29컷, s24 블랙 제외) ✓
- **촬영/편집 분리** — 생성=테이크(전부 4초+), 조립=편집. "컷==샷(생성 단위)" 가정 폐기,
  테이크 재사용 허용 (오너 교정: "1초짜리 컷 쓰지 말고 샷을 최소 4초 단위로") ✓

- 편집 리듬 = **원본 보존** (1초 컷도 테이크에서 오려 재현 — 잔여 확정 "니 말대로" 승인) ✓
- 시나리오 = `scenario.md` 초안 완료 — 스토리 비트 확정 반영: 희미한 "somebody help me"(~8-9s,
  s09 중) → 큰 "help me"+놀람·거울→세면대 시선(~17s, s12) → 비명+암전(47s, s24) →
  도플갱어(s26). **문안 최종 확인만 남음** ✓
- 다음 단계: 시나리오 확인 → BKM 테이크별 카메라 4요소 층 작성 → 이미지 스테이징 → 생성
