# 전편 카피 v2 — 입력 배선을 고치고 다시 잰다 (ORIGIN vs FRAMEFIX vs BKM2)

> **한 줄**: v1(전편 카피 묶음 실험)은 팔 두 개의 **입력이 결함**이라 무효가 됐다 — 제품판은 정본
> i2i가 아닌 무앵커 T2I 그림이, 사람판은 도입 플래시를 오분할한 테이크가 들어갔다. 둘을 고친
> 3팔로 같은 질문을 다시 잰다: "제품의 의도된 배선(ORIGIN) vs 프레임 고정 업그레이드(FRAMEFIX)
> vs 사람 고점(BKM2), 원본 카피에서 각각 어디까지 가는가."
>
> 설계일 2026-07-24 · 선행: [v1 무효 판정](../2026-07-23_full-copy-bundle/result.md) · 상태: **설계도 3장 오너 확인 대기**

## 1. v1이 무효가 된 이유 (이 실험의 존재 이유)

1. **제품판(BASE) 입력 결함**: 제품 writer의 샷 이미지 스테이지는 정본 에셋 매니페스트가 있으면
   i2i(정본 참조 편집)로 가도록 짜여 있으나, 그 매니페스트를 만들던 스테이지가 비주얼 축
   리디자인 때 삭제돼 현재는 **아무도 만들지 않는다**(개선 백로그 I9). 실험은 제품 코드를 그대로
   탔고, 그 결과 샷 20개가 서로 무관한 T2I로 생성돼 실사/유화/누아르가 뒤섞인 입력이 영상으로
   들어갔다. "현행 제품 실측"으로는 유효했으나, 오너 판정: 제품의 **의도된** 배선(artist 탭 정본
   i2i)이 살아있는 상태를 재야 의미가 있다.
2. **사람판(BKM) 테이크 계획 결함**: 원본 도입부(0~5.8초)는 1초 안팎 플래시 컷 8개(s01~s08)의
   빨리감기 몽타주다. v1은 컷마다 독립 4~5초 테이크를 할당했고, 따로 생성된 8개 세계를 1초 미만씩
   이어붙인 도입부는 붕괴했다. 생성 프레임의 구도도 원본과 동떨어졌다(예: s26 끌기 — 원본은
   인물이 좌하단에 작게, 생성본은 중앙에 크게. QC 게이트가 실효 없었음).

## 2. 팔 구성 (설계도 3장 — 각 문서가 발사 가능한 전체 페이로드를 담는다)

| 팔 | 한 줄 정의 | 설계도 |
|---|---|---|
| **ORIGIN** | "origin product pipeline" — 정본 세트(캐릭터 시트+배경 플레이트) × writer 샷 프롬프트 → i2i 시작 프레임 → 시작 프레임+모션 문장으로 영상. 제품의 의도된 배선 복원판 | [blueprint_origin.md](blueprint_origin.md) |
| **FRAMEFIX** | "frame fixed product pipeline" — ORIGIN과 동일하되 i2i를 **시작+끝 프레임 쌍**으로 만들어 둘 다 영상 모델에 못박는 제품 업그레이드 가설판. 유일 변수 = 끝 프레임 | [blueprint_framefix.md](blueprint_framefix.md) |
| **BKM2** | 사람 고점 재설계판 — 도입 플래시는 신규 테이크 금지·후반 동일 구도 테이크에서 오려 쓰기, 프레임 전량 재생성(원본 프레임 대조 QC 강제), 구도 지시 정밀화 | [blueprint_bkm.md](blueprint_bkm.md) |

**대조군**: 원본과 상한판 R(원본 프레임 직입력)은 v1 산출을 그대로 재사용한다 — R의 입력은
원본 프레임이라 v1의 입력 결함과 무관하다.

## 3. 공통 재료·불변 조건

- **writer 산출 재사용**: v1 BASE 실행분(20샷 — 샷 분해·카메라·이미지/모션 프롬프트,
  `../2026-07-23_full-copy-bundle/assets/arm-base/shots.json`). ORIGIN·FRAMEFIX가 **같은 20샷**을
  쓰므로 두 팔의 차이는 오직 프레임 공급 형태다. LLM 재실행 없음(비용 0, 비교 성립).
- **정본 세트**: 캐릭터 정본 `../2026-07-23_character-canon/assets/identity_ref.jpg` + 빈 방
  플레이트 `../2026-07-23_input-format/assets/plates/src_empty_wide.jpg`.
- **모델·레인 통일 (v1 확정 유지)**: 영상 전 팔 Seedance 2.0 · 힉스필드 레인. 이미지 fal
  `openai/gpt-image-2/edit`.
- **시나리오·원본**: v1과 동일 (`../2026-07-23_full-copy-bundle/scenario.md`, 원본 67초·30컷).

## 4. 가설 (사전 고정)

- **H1**: ORIGIN은 v1 BASE 대비 신원·스타일 드리프트가 소멸한다 (배선 복원 효과의 실측).
- **H2**: FRAMEFIX는 ORIGIN 대비 샷 내부 카메라 폭주·동작 이탈이 줄어든다. (입력 포맷 실험에서
  "그림 못박기" 가설이 기각됐으나 그때는 무앵커 조건 — 정본 앵커 위에서 재검증.)
- **H3**: BKM2는 두 제품 팔에 오너 블라인드 구간 순위 명확 우위다. 격차 크기 = writer 연출층
  개선 여지 (v1의 원래 질문 승계).

## 5. 판정 (사전 고정)

v1 방식 승계: 오너 블라인드, 구간별(도입 0~6s / 립 6~19s / 조사 19~47s / 결말 48~66s) 순위 +
전체 순위. 비교 대상이 5편(원본·R·ORIGIN·FRAMEFIX·BKM2)이므로 모자이크 배치(2×2 두 세트 or
3×2 한 판)는 조립 시점에 정하고 키 파일로 라벨 관리. 규칙: **BKM2가 제품 두 팔을 과반 구간에서
이기면** 사람판 묶음 배선 상신, **FRAMEFIX가 ORIGIN을 과반 구간에서 이기면** 끝 프레임 생성을
제품 배선 후보로 상신(I8과 연동).

## 6. 순서·예산

설계도 3장 오너 확인 → ORIGIN 프레임 스테이징(20 i2i) → FRAMEFIX 끝 프레임(20 i2i, ORIGIN 프레임
참조 의존) → BKM2 스테이징+QC → 영상 발사(3팔) → 조립·모자이크 → 블라인드 판정.

영상 예산 개산: ORIGIN ~74s + FRAMEFIX ~74s + BKM2 ~100s ≈ 250s × 4.6크레딧/초 ≈ **~1,150크레딧**
(+QC 재생성 여유 15%). 세부는 각 설계도의 실행 스펙 참조. 이미지(fal)는 별도 과금 ~80콜.
