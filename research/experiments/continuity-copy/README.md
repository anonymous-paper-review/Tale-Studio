# continuity-copy — 연출 연속성·레퍼런스 카피

**이 트랙이 쫓는 질문**: 컷이 나뉘어 있어도 "이어져 보이는" 연출을, 우리 파이프라인으로 만들 수 있나?
오너의 문제 제기에서 출발했다 — "모든 샷이 컷 편집 수준으로 뚝뚝 끊긴다. 같은 장소·같은 상황의 연속 샷이면
샷 간 연속성이 굉장히 중요하다."

## 타임라인

1. **레퍼런스 해석** — 잘 이어지는 실제 영상(거울 앞 소녀, 67초)을 프레임 단위로 뜯어봤다.
   핵심 발견: 연속감은 롱테이크가 아니라 **사건당 3~4샷 클러스터 + 동작 이어받기 + 마스터 후렴 + 불변 앵커**에서 온다.
   컷 수는 우리와 비슷했다(67초에 30컷). → [`../../references/2026-07-22-girls-in-mirror-continuity.md`](../../references/2026-07-22-girls-in-mirror-continuity.md)
2. **세탁방 시제품** — 해석한 연속성 문법으로 60초 영상 하나를 실제로 만들어봤다(새벽 코인세탁방).
   → [`2026-07-23_laundromat-proto/result.md`](2026-07-23_laundromat-proto/result.md)
3. **오너 평가** — 클러스터 구조는 맞으나 연속감이 아직 미달, 불필요한 샷·카메라/시선 불안정.
   문법만으로는 부족 → 레퍼런스를 **완벽히 카피**해 재현 가능성부터 증명하는 쪽으로 전환(E9d-copy).
4. **캐릭터 정본** — 카피의 공통 재료로 쓸 인물 신원을 확정했다(텍스트 후보는 폐기, 레퍼런스 정면 1장 채택).
   → [`2026-07-23_character-canon/notes.md`](2026-07-23_character-canon/notes.md)
5. **입력 포맷 실험 설계** — 같은 콘티(정답지)를 여러 입력 방식으로 재현시켜 "어떤 입력 포맷이 연속성을
   가장 잘 만드나"를 비교하는 실험을 설계했다. 콘티 6샷 + 방식 4종 설계도, 오너 검토 대기.
   → [`2026-07-23_input-format/conti.md`](2026-07-23_input-format/conti.md)
