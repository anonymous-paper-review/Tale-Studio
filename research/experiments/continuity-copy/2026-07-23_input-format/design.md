# E9d-copy 입력 포맷 실험 설계 — "영상 생성기에 무엇을 넘겨야 연출이 유지되는가"

> **한 줄 요약: 같은 장면을 네 가지 "넘겨주는 방식"으로 만들어 원본과 나란히 놓고, 어느 방식이
> ① AI 특유의 제멋대로 카메라 움직임 ② 샷 사이 인물·공간 깨짐 ③ 뜬금없는 샷을 가장 잘 없애는지
> 가린다.** 완벽 카피 본작업은 이 실험의 승자 방식으로 진행한다.
>
> 설계일 2026-07-23 · 오너 지시("i2v에 들어갈 소스가 잘못된 것 같다 — 돌리기 전에 점검") · 상태: 오너 확정 대기

## 1. 왜 이 실험이 먼저인가 — 시제품 실패의 분해

코인세탁방 시제품(60초)에 대한 오너 관찰:

> "AI가 만든 특유의 카메라 무브와 쓸데없는 샷이 너무 많음. 뜬금없이 나오고 그 뒤에 갑자기 캐릭터가
> 나오는, 심지어 컨시스턴시가 다 깨지면서."

원인을 셋으로 분해하면 전부 **영상 모델에 넘기는 입력의 문제**다:

| 증상 | 원인 | 이번 실험에서 고치는 장치 |
|---|---|---|
| 카메라가 제멋대로 움직임 | 참조형 영상 모델(시작 그림을 "참고만" 함) + 움직임의 도착점을 안 알려줌 | 엄격한 시작 프레임 고정 + **끝 프레임 명시** |
| 샷 사이 인물·공간 깨짐 | 앵커가 텍스트 한 줄뿐 — 샷마다 인물·공간을 새로 해석 | 캐릭터 정본·배경 자산을 **이미지로** 전달 |
| 뜬금없는 샷, 갑작스러운 인물 등장 | 샷별 완전 독립 생성 + 저작이 사물 인서트를 과다 배치 | 콘티를 레퍼런스에서 가져와 통제(카피이므로) + 샷 묶음 연결 |

## 2. 공통 통제 (모든 방식에 동일)

- **재현 대상**: 레퍼런스 "거울 영상"의 대표 구간 1개 — **제안: 립 바르기 구간(약 5.8~16.2초, 샷 3개:
  거울 정면 5.5초 홀드 → 앵글 전환 → 뒤통수/거울) + 마스터 와이드 1샷 + 인서트 1샷 = 총 5샷 약 13초.**
  한 구간 안에 홀드·클러스터·동작 이어받기·인서트·마스터가 전부 들어 있어 판별력이 가장 높다.
- **콘티(정답지)**: 원본에서 샷별 시작/끝 프레임·길이·카메라 움직임·동작을 추출해 고정. 모든 방식이
  같은 콘티를 재현한다 (저작 변수 제거 — 순수하게 "입력 포맷"만 비교).
- **캐릭터 정본**: `../2026-07-23_character-canon/assets/identity_ref.jpg` (레퍼런스 10.9초 정면 1장 — 신원 전파
  검증 완료).
- **영상 모델**: 선행 스크리닝(§4-가)에서 뽑은 엄격 I2V 1개로 통일.
- **평가**: §5의 사전 고정 기준. 원본 포함 블라인드 나란히 보기(오너) + 프레임 대조 정량 카운트(나).

## 3. 비교할 방식 4개 + 상한 대조군

### A안 — 자산 + 연출 텍스트 (우리 워크플로우 강화판) 〔오너 시나리오 1〕
- **넘기는 것**: ① 캐릭터 시트(정본 1장에서 다각도 4컷을 미리 생성해 한 장으로 합침) ② 배경 플레이트
  (빈 화장실을 앵글별로 준비) ③ writer식 연출 텍스트(카메라·동작).
- **만드는 법**: 샷별 첫 프레임 = 편집 모델(시트+배경 참조, 구도 지시) → 엄격 I2V(첫 프레임 + 움직임 텍스트).
- **가설**: 자산을 이미지로 주면 인물·공간 깨짐은 잡히지만, 움직임 도착점이 없어서 카메라 무브 문제는
  남을 것.

### B1안 — 시작+끝 프레임 쌍 (SNS 방식의 핵심 분리판) 〔오너 시나리오 2의 ①〕
- **넘기는 것**: 샷마다 **시작 프레임과 끝 프레임 2장**(둘 다 편집 모델로 생성 — 끝 프레임이 동작·카메라의
  도착점을 그림으로 못박음) + 짧은 움직임 텍스트.
- **만드는 법**: 시작/끝 프레임 지원 영상 모델로 두 그림 사이를 보간 생성.
- **가설**: 도착점이 그림으로 고정되므로 **AI 임의 무브가 원천 차단**된다 — 이 실험의 유력 승자 후보.

### B2안 — 연출 템플릿 시트 한 장 (SNS 유행 원형) 〔오너 시나리오 2의 ②〕
- **넘기는 것**: 캔버스 한 장에 표 레이아웃으로 [샷 번호 | 시작 그림 | 끝 그림 | 연출 노트]를 전부 박은
  **거대한 한 장짜리 연출 시트** + "이 시트대로 만들어라" 지시.
- **만드는 법**: 시트는 우리가 코드로 합성 → 시트 해석이 가능한 영상 모델에 통째로 입력.
- **가설**: 되면 파이프라인이 가장 단순해진다(시트 1장=클립 N개). 리스크: 모델이 시트를 "화면에
  그대로 그려버리는" 오작동 — 이것 자체가 검증 포인트.

### C안 — 클러스터 체이닝 (제3의 방법 제안) 〔오너 "또다른 방법?" 슬롯〕
- **넘기는 것**: B1과 같되, 같은 사건 묶음(클러스터) 안에서는 **앞 샷 생성 결과의 실제 마지막 프레임을
  추출해 다음 샷의 시작 프레임 생성에 반영**(동작 위상·소품 상태가 실물로 이어짐).
- **가설**: 컷을 관통하는 동작(레퍼런스 해석의 핵심 장치)이 가장 충실히 재현된다. 대가: 순차 생성이라
  느리고, 앞 샷의 오류가 뒤로 전파될 위험.

### R — 상한 대조군 (비교용, 방식이 아님)
- **넘기는 것**: 원본 영상의 실제 프레임을 시작(B1은 시작+끝) 프레임으로 그대로 사용 → 엄격 I2V.
- **목적**: 이미지 생성 계층의 손실을 0으로 만든 "영상 모델 성능의 상한선". A~C가 R에 얼마나
  근접하는지로 "이미지 계층 손실"과 "영상 계층 손실"을 분리 측정한다.

## 4. 선행 작업 2건 (방식 비교 전에 확정)

**가. 엄격 I2V 모델 스크리닝** — 지금 제품 배선 3종은 전부 참조형(개선 목록 I8). fal에서 시작 프레임을
고정하는 후보 2~3개(끝 프레임 동시 지원 여부 포함)를 골라, 같은 시작 프레임+같은 움직임 텍스트로
1샷씩 생성 → ① 첫 프레임 유지율(원본 그림과 픽셀 유사도) ② 지시한 움직임만 하는지 ③ 시작+끝 쌍
지원 여부로 1개 선정. (6콜 내외)

**나. 콘티 추출** — 대표 구간 5샷의 시작/끝 프레임·길이·움직임 기술을 원본에서 추출해 `콘티표`로 고정.
(생성 0콜 — 이미 추출된 컷 데이터 재사용)

## 5. 평가 기준 (실행 전 고정)

**정량 (내가 프레임 대조로 카운트, 방식당·샷당):**
1. 임의 카메라 무브 수 — 원본/콘티에 없는 움직임이 나온 횟수 (오너 1순위 불만)
2. 컨시스턴시 붕괴 수 — 인물 외형·의상·공간·소품이 직전 샷과 달라진 횟수
3. 구도 재현 충실도 — 생성 첫 프레임 vs 원본 해당 프레임 (구도 요소 체크리스트)
4. 도착점 정확도 — 클립 마지막 프레임이 의도한 끝 상태에 도달했는가
5. 파이프라인 비용 — 방식당 생성 콜 수·단계 수 (제품화 난이도)

**정성 (오너 블라인드):** 원본 + 방식 4개(+R)를 라벨 없이 나란히 → 연속감·연출 충실 순위 매기기.

**판정 규칙**: 정량 1·2에서 최저 + 오너 순위 상위인 방식을 카피 본작업 표준으로 채택. B2가 오작동하면
그 사실만 기록하고 탈락(잔여 판단 오염 방지).

## 6. 비용·순서

| 단계 | 내용 | 생성 규모 |
|---|---|---|
| 선행-가 | I2V 스크리닝 | 클립 ~6 |
| 선행-나 | 콘티 추출 | 0 |
| 방식 A | 시트+플레이트+프레임 5+클립 5 | 이미지 ~12 · 클립 5 |
| 방식 B1 | 프레임 10+클립 5 | 이미지 ~12 · 클립 5 |
| 방식 B2 | 시트 합성+클립 | 이미지 ~12 · 클립 1~5 |
| 방식 C | B1+체이닝 재생성 | 이미지 ~8 · 클립 5 |
| 합계 | | **이미지 ~45 · 클립 ~26** (시제품과 비슷한 규모) |

순서: 가 → 나 → A·B1·B2·C 병렬 생성 → 5-way 블라인드 → 판정 → 승자 방식으로 완벽 카피 본작업.

## 7. 오너 확정 필요 항목

1. 팔 구성 — A·B1·B2·C(+R 대조군) 이대로 좋은가, 뺄/더할 것 있는가
2. 대표 구간 — 제안(립 바르기 클러스터+마스터+인서트, 13초 5샷) 동의 여부
3. 실행 승인 — 승인 시 선행-가(모델 스크리닝)부터 착수

## 8. SNS 실물 검증 반영 (v2 개정, 2026-07-23)

오너가 제공한 SNS 워크플로 실물(스킬 3종 + 시트 실물 — 해석: [`../../../references/2026-07-23-sns-seedance-method.md`](../../../references/2026-07-23-sns-seedance-method.md))을
분석해 설계를 4곳 보정했다:

1. **전 팔 공통 "텍스트 계약" 추가** — 지금의 앵커 한 문장을 3층으로 확장: ① 연속성 바이블(의상
   LOCKED·빛 방향·180° 라인·시그니처 소품) ② 드리프트 네거티브 배터리(의상 플리커·그림자 방향
   뒤집힘·낮밤 점프·다중 얼굴·플라스틱 피부 금지 문장) ③ 샷별 결말 상태(끝나는 순간의 자세·카메라·
   조명 명문화). 팔 간 차이는 여전히 "이미지를 어떻게 주느냐"만이므로 통제 변수 원칙 유지.
2. **B2 재정의** — 원안("시트를 영상 모델에 통째 입력")은 실제 SNS 용법이 아니었다. SNS의 시트는
   설계 시각화 + **프레임 수확**용(한 번의 이미지 생성 안에 여러 컷이 그려져 컷 간 일관성이 공짜).
   B2 본체를 "콘티 시트 1장 생성 → 셀 잘라 시작 프레임으로 수확 → 이후 B1과 동일"로 교체하고,
   통째 입력은 β 1회 시도로 강등(오작동 여부만 기록). 상세: blueprint-b2.md 재정의 절.
3. **선행 스크리닝에 Seedance 계열 필수 포함** — "픽셀 단위 첫 프레임 고정"을 명시하는 유일 후보군.
   영상 시드 고정 지원 여부를 스크리닝 항목에 추가.
4. **카메라 4요소 표기**(렌즈·앵글/구도·무브·광학)는 이번 콘티가 전 샷 고정 카메라라 실험 무관 —
   writer 본체 개선 후보로 백로그 이관(I9 후보).
