# Overview

본 시스템은 사용자는 영화감독, 시나리오작가 등 시네마관련 종사자이다. (B2B)

단순한 텍스트 입력(Story) 및 자동화된 제작 과정을 통해 편의성 및 비용 절감을 추구한다.

차별화 기술은 **전문적인 영화 촬영 기법(Rag)이 적용된 고품질 AI 비디오 생성** 엔진이다.

나머지 기술은 다른 플랫폼이 제공하는 것과 동일한 수준으로 제공하는 것을 목표로 한다.


## 용어 정의

| 용어 | 정의 |
| --- | --- |
| **Story** | 전체 스토리 (1개 영상 프로젝트) |
| **Scene** | 시간/장소가 동일한 연속 장면 (여러 Shot으로 구성) |
| **Shot** | 최소 생성 단위 = API 호출 1회 = ~5초 영상 |

**MVP 목표**: 약 8분 영상 = 4개 Scene = 24~48개 Shot 

- 기승전결 4개 씬에 2분(120초)씩으로  가정해서 8분.
- 1개 씬마다 길이는 다를 수 있음.
- 1개 씬마다 9~18개의 샷(45초~90초)으로 구성 가정

## UX 플로우

[UX.pdf]

1. Ground Page
    - 사용자 텍스트 및 이미지 입력
        - 스토리 글 저장, 캐릭터 이미지 (Max 3장) 저장
    - Writing 진입
        - Next Button
    - 경쟁사 참고 UX(oii_1.png)

        
2. Writing Page
    - Level 1
        - Level 1 동작
        - 각 씬마다 프롬프트 글 수정 가능
        - Result: 순서 및 도식화된 씬과 해당 프롬프트 글 (*** MVP에서는 4개씬, 기능전결)
    - Storyboard 진입
        - 씬별로 Next Button
    - 경쟁사 참고 UX(oii_2.png)

        

1. Storyboard Page (씬별)
    - Level 2-1 + Level 3 동시 동작
        - Level 2-1(씬 프롬프트 → 샷 프롬프트) 동작
        - 샷에 필요한 요소 자동 추천 (촬영기법, Start-End, Only Start, Only End, Add Characters)
    - Level 2-2
        - Level 2-2(샷 프롬프트 → 샷 이미지) 동작
        - 이미지에 대한 촬영 기법 커스터마이징 (wide, cu, ots 등 카메라, 조명)
    - Level 2-3 + Level 3 동시 동작
        - 영상 생성
        - 앞으로 돌아가서 다시 생성 가능
    - 경쟁사 참고 UX(oii_3.png)
