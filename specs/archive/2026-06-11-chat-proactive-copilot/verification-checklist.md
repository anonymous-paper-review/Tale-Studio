# chat-proactive-copilot — 브라우저 검증 체크리스트 (몰아서 검증용)

> 각 Phase 코드 작성 후 누적. 검증 완료 시 tasks.md 해당 `[c]` → `[x]`.
> 공통 준비: `pnpm dev` → 로그인 → producer에서 스토리 입력 → "Hand over to Concept Artist".

## Phase 1 — 프로액티브 넛지

- [ ] **넛지 등장**: artist 진입 → 자동생성(캐릭터/배경) 완료 후 ~1.5s 뒤 채팅 패널에
      "캐릭터 N명·배경 M개가 모두 준비됐어요 — Director로 갈까요?" 버블 등장.
- [ ] **승인 동작**: "Director로 가기" 클릭 → director 스테이지로 이동 + 제안 사라짐.
- [ ] **거부 동작**: "나중에" 클릭 → 제안 사라짐, 아무 일 없음.
- [ ] **재발사 방지**: 거부/승인 후 artist로 재진입 → 같은 넛지 다시 안 뜸 (새로고침 전까지).
- [ ] **비용 무발생**: 넛지 등장/승인/거부 어디서도 새 이미지·영상 생성이 일어나지 않음
      (네트워크 탭에 generate 호출 없음).
- [ ] **카피 정확성**: 배경이 0개인 프로젝트면 "캐릭터 N명이 모두 준비됐어요"로 자연스럽게 표시.

## Phase 2 — 완료 알림 (크로스스테이지 배지 + 채팅 메시지)

준비: artist에서 이미지 생성이 도는 동안(또는 director에서 영상 생성 중) **다른 stage로 이동**.

- [ ] **배지 등장**: artist에서 생성이 진행 중인 채로 director로 이동 → 생성이 끝나면 사이드바의
      **artist 아이콘에 숫자 배지**가 뜬다. (반대로 director 영상 생성 중 artist로 가면 director에 배지)
- [ ] **같은 stage면 무알림**: 생성 중인 stage에 그대로 머물면 배지/메시지 안 뜸(보고 있으니 불필요).
- [ ] **채팅 메시지**: 완료 시 채팅에 "✓ … 생성이 완료됐어요. … 탭에서 확인하세요" 모델 메시지(스테이지 뱃지 포함).
- [ ] **스로틀**: 이미지 여러 장이 연달아 완료돼도 채팅 메시지는 10초에 1개로 합쳐짐(배지 숫자는 누적).
- [ ] **배지 클리어**: 배지가 뜬 stage로 이동 → 배지 사라짐.

## Phase 3 — 멀티유저 쿼터

> 단일 유저로는 상한(30) 도달이 어려워 코드 검증만으로 충분(tsc/lint clean). 선택적 수동 확인:
- [ ] (선택) `MAX_QUEUED_JOBS_PER_USER`를 임시로 1~2로 낮춰 생성 2개 연속 → 두 번째가 429 +
      "생성 대기열이 가득 찼어요" 에러 표시되는지 확인 후 원복.
- [ ] 정상 사용(상한 미만)에선 생성이 평소처럼 동작(가드가 막지 않음).

## Phase 4 — 누락 감지 넛지 (director)

- [ ] **갭 넛지 등장**: 샷에 캐릭터/배경 참조가 없거나 스토리보드 미생성인 director 프로젝트 진입 →
      ~2s 후 채팅에 "채워두면 좋을 항목이 N건 있어요: • …" informational 제안 등장.
- [ ] **action 없음**: 이 제안은 "나중에"(dismiss)만 있고 생성을 트리거하지 않음.
- [ ] **갭 없으면 무알림**: 모든 샷이 참조+스토리보드 완비면 넛지 안 뜸.
- [ ] **재발사 방지**: dismiss 후 재진입 시 다시 안 뜸(세션 내).

## Phase 5 — writer 채팅 제거 / parity

- [ ] writer는 사이드바에 없으므로 직접 영향 없음 — 회귀만 확인: producer/artist/director 채팅 정상 동작.
- [ ] (문서) `parity-audit.md` 의 갭 목록이 실제와 맞는지 1회 훑어보기(후속 change 입력).
