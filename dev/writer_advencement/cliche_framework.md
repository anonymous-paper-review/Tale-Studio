# 클리셰 프레임워크: 교차 관심사(Cross-Cutting Concern)

> 작성일: 2026-04-16
> 위치: S축, V축, C(서사) 전체를 관통하는 독립 레이어
> 핵심: 클리셰는 특정 카테고리 항목이 아니라, 모든 창작 결정에 적용되는 판단 기준

---

## 1. 클리셰의 7가지 차원

클리셰는 단일 현상이 아니라 7개 차원이 겹친 복합체.

```
      [1] 관객 기대       수신자가 패턴을 인식하고 예측하는 문제
      [2] 창작 관성       생산자가 탐색 없이 첫 번째 해결책을 택하는 문제
      [3] 소통 압축       공유 어휘로 빠르게 의미를 전달하는 기능
      [4] 사회/문화       고정관념을 재생산하고 정상화하는 위험
      [5] 시간/역사       시대와 문화에 따라 경계가 이동하는 상대성
      [6] 구조/형식       서사 필수 구조와 특정 실현의 경계
      [7] 상호텍스트       레퍼런스, 오마주, 패러디, 카피, 클리셰의 스펙트럼
```

---

### [1] 관객 기대 (Reception Dimension)

**정의**: 관객의 축적된 시청 경험이 패턴 인식 → 예측 → 확인/실망을 만듦.

**메커니즘**:
```
첫 노출:   패턴 학습    → "이런 것도 있구나" (신선)
반복 노출:  패턴 인식    → "아 이거 알아" (친숙)
과다 노출:  패턴 예측    → "다음에 이렇게 되겠지" (예측 가능)
포화:      패턴 거부    → "또 이거야" (클리셰)
```

**관객 경험 수준별 반응 차이**:

| 관객 유형 | "영웅이 마지막에 이긴다" 반응 |
|----------|--------------------------|
| 어린이 | "우와 이겼다!" (신선) |
| 일반 관객 | "이길 거란 건 알지만 어떻게?" (관습, 편안) |
| 장르 마니아 | "또 이기네. 어떻게 이기는지도 뻔해" (클리셰) |
| 비평가 | "이기는 패턴을 어떻게 비틀었나?" (메타적 감상) |

**시사점**: 같은 패턴이 관객에 따라 신선/관습/클리셰 모두 될 수 있음. 타겟 관객의 경험 수준이 클리셰 판단의 기준선.

**파이프라인 적용**: 프로젝트 설정(S0)에서 타겟 관객 정의 → 해당 관객의 장르 경험 수준 추정 → 클리셰 감도 기준선 설정.

---

### [2] 창작 관성 (Production Dimension)

**정의**: 작가가 가장 먼저 떠오른 해결책을 채택하고 더 나은 대안을 탐색하지 않는 것. 관객과 무관한 **창작 프로세스의 문제**.

**메커니즘**:
```
문제: "캐릭터의 슬픔을 어떻게 표현하지?"

관성적 해결:  비를 내린다 (1초 만에 떠오름, 검증 없이 채택)
탐색적 해결:  비를 내린다 → 맑은 날 장례식은? → 캐릭터가 웃는 건?
             → 아무 반응 없는 건? → 3번째나 4번째 아이디어 채택
```

**왜 발생하나**:
- 시간 압박 (마감)
- 인지 편향 (첫 아이디어에 고정)
- 참조 과다 (최근 본 작품의 무의식적 모방)
- "작동하는 것"에 대한 안전 선호

**관객 기대와 다른 점**:
관객이 한 번도 본 적 없는 패턴도 창작 관성의 결과일 수 있음.
반대로, 관객이 많이 본 패턴이라도 의도적 선택이면 관성이 아님.

```
관성 = 탐색 없이 택한 첫 아이디어
관습 = 알고서 택한 검증된 패턴
클리셰 = 관성 + 관객 포화의 교집합
```

**파이프라인 적용**: AI가 스토리 요소를 생성할 때 "첫 번째 출력"을 기본값으로 쓰지 않고, 3개 이상의 변형을 생성 → 가장 예측 불가능한 것을 제안 (자가 검증).

---

### [3] 소통 압축 (Semiotic Efficiency)

**정의**: 클리셰/관습은 **공유된 문화 어휘**로서, 복잡한 의미를 최소 시간에 전달하는 효율적 기호.

**이것은 클리셰의 긍정적 기능이다.**

```
"장례식에 비"     = 대사 없이 슬픔 전달 (1초)
"틱탁 소리"       = 대사 없이 시간 압박 (1초)
"어두운 골목"     = 대사 없이 위험 (1초)
"빨간 드레스"     = 대사 없이 매력/위험 (1초)
"깨진 가족사진"   = 대사 없이 이혼/상실 (1초)
```

**왜 이것이 중요한가 (Tale Studio 특수 사정)**:

```
AI 비디오 생성 프롬프트 = 150자 제한

150자 안에 넣어야 하는 것:
  - 캐릭터 설명
  - 행동
  - 감정
  - 환경
  - 카메라
  - 조명
  - 분위기

→ 효율적 기호(관습)가 필수
→ "rain, funeral, grief" = 3단어로 환경+감정+맥락 전달
→ 관습을 피하면 150자로 아무것도 전달 못 함
```

**관습의 압축 기능을 유지하면서 클리셰를 피하는 법**:

| 전략 | 설명 | 예시 |
|------|------|------|
| **기호 사용 + 변형 추가** | 기본 기호로 빠르게 소통 후 하나를 비틂 | "funeral, rain, but she's laughing" |
| **기호 조합** | 보통 같이 안 쓰는 기호를 결합 | "dark alley + children playing" (위험+순수) |
| **기호 부재** | 기대되는 기호를 의도적으로 제거 | "funeral, blazing sunshine, silence" |
| **기호 반전** | 기호의 의미를 뒤집음 | "rain = joy" (웃으며 비 맞기) |

**핵심 원리**: 관습의 소통 효율을 취하되, 하나의 요소를 틀어서 클리셰에서 탈출. 100% 신선하려고 하면 소통 자체가 실패.

---

### [4] 사회/문화 (Ideological Dimension)

**정의**: 특정 클리셰가 고정관념을 재생산하고, 해로운 서사를 정상화하는 문제. 관객이 기대하고 만족하더라도 사회적으로 해로울 수 있음.

**캐릭터 클리셰 (해로운)**:

| 클리셰 | 문제 | 실제 의미 |
|--------|------|----------|
| Magical Negro | 흑인 캐릭터가 백인 주인공 돕기 위해만 존재 | 흑인의 주체성 박탈 |
| Women in Refrigerators | 여성이 남성 동기부여를 위해 죽음/피해 | 여성을 플롯 도구로 환원 |
| Manic Pixie Dream Girl | 여성이 남성의 인생을 바꿔주기 위해 존재 | 여성의 독립적 서사 부재 |
| 교활한 동양인 | 아시아 캐릭터가 신비롭고 수상하게 묘사 | 인종적 타자화 |
| Gay Best Friend | 성소수자가 주인공의 조언자/엔터테이너로만 | 성소수자의 고유 서사 박탈 |
| Disabled Inspiration | 장애인이 비장애인에게 영감을 주기 위해 존재 | 장애의 도구화 |

**서사 클리셰 (해로운)**:

| 클리셰 | 재생산하는 이데올로기 |
|--------|-------------------|
| "노력하면 성공" | 구조적 불평등 은폐 (능력주의 신화) |
| "선한 폭력" | 주인공의 폭력은 정당하다는 전제 |
| "사랑이 모든 걸 해결" | 구조적/심리적 문제의 감정적 축소 |
| "경찰은 항상 옳다" | 권력 기관 무비판적 정당화 |
| "아름다움 = 선함" | 외모와 도덕의 연결 |
| "문명 vs 야만" | 문화적 우월주의 |

**[1]과 [4]의 차이**:
```
[1] 관객 기대: "이 패턴 지루해" (미학적 판단)
[4] 사회/문화: "이 패턴 해로워" (윤리적 판단)

겹치는 경우: 관객이 지루해하면서 동시에 해로운 패턴
겹치지 않는 경우: 관객이 즐기지만 해로운 패턴 (가장 위험)
```

**파이프라인 적용**: AI가 캐릭터/씬을 생성할 때 해로운 클리셰 목록과 자동 대조. CRITICAL 플래그로 경고.

---

### [5] 시간/역사 (Temporal Dimension)

**정의**: 클리셰의 경계는 시대와 문화에 따라 이동. 동일한 패턴이 어디서/언제냐에 따라 신선/관습/클리셰가 됨.

**시대별 이동**:

```
시간축 →

호메로스: "영웅 귀환"           = 원형(archetype), 새로움
셰익스피어: "운명적 연인"       = 비극의 발명
1940년대: "하드보일드 탐정"     = 장르 확립
1970년대: "하드보일드 탐정"     = 관습
2000년대: "하드보일드 탐정"     = 클리셰 (또는 오마주)
2020년대: "하드보일드 탐정"     = 레트로/노스탤지 (의도적이면)
```

**문화별 차이**:

| 패턴 | 한국 | 미국 | 인도 |
|------|------|------|------|
| 재벌+가난한 주인공 | 클리셰 (K드라마 과포화) | 비교적 신선 | 보편적 (신분 서사 전통) |
| 슈퍼히어로 오리진 | 신선할 수 있음 | 클리셰 (MCU 피로) | 신화적 전통과 연결 |
| 가족 승인 결혼 | 드라마 관습 | 다소 이국적 | 필수 서사 요소 |
| 복수극 | 관습~클리셰 | 액션 관습 | 장르적 필수 (마살라) |

**시사점**: "이것은 클리셰인가?"는 절대적 질문이 아님. "누구에게, 언제, 어디서 클리셰인가?"가 올바른 질문.

**파이프라인 적용**: 프로젝트 S0에서 타겟 시장/문화권 설정 → 해당 문화의 클리셰 사전 참조.

### [6] 구조/형식 (Structural Dimension)

**정의**: "서사의 뼈대"와 "특정 실현(살)"의 경계. 뼈대는 클리셰가 될 수 없고, 살만 클리셰가 될 수 있음.

```
뼈대 (구조적 필수, 클리셰 아님):
  ├── 주인공이 도전에 직면한다
  ├── 갈등이 에스컬레이션한다
  ├── 클라이맥스에서 핵심 질문에 답한다
  ├── 캐릭터가 선택을 하고 결과를 맞는다
  └── 시작과 끝이 있다

살 (특정 실현, 클리셰 가능):
  ├── "마지막 1초에 폭탄 해제" (도전 직면의 한 실현)
  ├── "악당이 계획을 설명" (에스컬레이션의 한 실현)
  ├── "영웅이 연설로 군중을 감동" (클라이맥스의 한 실현)
  └── "죽은 줄 알았던 캐릭터 부활" (결과의 한 실현)
```

**경계 판단 기준**:

| 질문 | "뼈대"면 | "살"이면 |
|------|---------|---------|
| 이것 없이 이야기가 성립하나? | 성립 안 함 | 성립함 (다른 살로 교체 가능) |
| 대안이 존재하나? | 대안 없음 (구조적 필수) | 대안 다수 존재 |
| 추상화하면 무엇인가? | "갈등" "선택" "변화" | "폭탄" "연설" "부활" |

**위험한 혼동**:
```
"3막 구조는 클리셰야" → 틀림. 3막 구조는 뼈대.
"3막 구조에서 2막 끝에 All Is Lost는 클리셰야" → 가능. 특정 실현이므로.
"캐릭터 아크는 클리셰야" → 틀림. 아크는 뼈대.
"속죄 아크에서 희생으로 속죄하는 건 클리셰야" → 가능. 특정 실현이므로.
```

**파이프라인 적용**: AI가 구조(뼈대)를 제안할 때 클리셰 경고를 하면 안 됨. 구조 안의 실현(살)을 제안할 때만 클리셰 검사.

---

### [7] 상호텍스트 (Intertextual Dimension)

**정의**: 모든 작품은 이전 작품들과 대화함. 같은 "가져오기"가 의도와 실행에 따라 다르게 판단됨.

**스펙트럼**:

```
의도적 ←───────────────────────────→ 무의식적
품질높음 ←───────────────────────────→ 품질낮음

        레퍼런스   오마주    패러디    카피     클리셰
의도:    알고 인용  존경 표현  희화화   베끼기   모르고 반복
관객인지: 알아보길기대 몰라도됨 알아야작동 알면실망 어디서왔는지모름
판단:    칭찬       존경     유머/비평  비판    게으름
```

**같은 패턴, 다른 판단**:

| 패턴 | 타란티노가 하면 | 무명 감독이 하면 |
|------|-------------|--------------|
| 멕시칸 스탠드오프 | "세르지오 레오네 레퍼런스! 영화 사학자!" | "서부극 클리셰. 신선함 없음" |
| 트렁크 샷 | "타란티노 시그니처!" | "타란티노 따라하기" |
| 비선형 구조 | "펄프 픽션의 혁신!" | "펄프 픽션 모방" |

**이것이 불공평해 보이지만 핵심은**:
```
레퍼런스 = "이 작품을 알고, 왜 가져오는지 안다" + 새로운 맥락 추가
클리셰  = "이 패턴이 어디서 왔는지 모르고, 그냥 씀" + 새로운 맥락 없음

차이 = 의식적 선택 + 변형/맥락 추가 여부
```

**파이프라인 적용**: AI가 장르 관습을 사용할 때 "출처 인식"을 메타데이터로 기록. "이 패턴은 [X]에서 유래하며, [Y]로 변형됨" — 출처 모르고 쓰는 것 방지.

## 2. 클리셰 라이프사이클

모든 패턴이 거치는 6단계:

```
[발명] → [모방] → [관습] → [클리셰] → [전복 대상] → [노스탤지/복고]
  │        │        │        │           │              │
 1회      N회     기대됨    지루함       메타적 활용     자각적 귀환
 최대     여전히   충족감    "또?"       "알면서 즐김"   "옛날엔 좋았지"
 효과     효과적
```

**사례로 보는 라이프사이클**:

```
"주인공 죽음 → 부활" 패턴:

발명:    성경의 부활 (원형)
모방:    판타지/신화에서 차용
관습:    슈퍼히어로 장르의 기대 요소 (죽었다 돌아옴)
클리셰:  MCU에서 모든 캐릭터가 죽었다 부활 → 죽음의 무게 상실
전복:    "어벤져스 엔드게임" — 진짜 죽이고 되돌리지 않음
복고:    아직 도래하지 않음 (부활이 다시 신선해지려면 시간 필요)
```

```
"반전 엔딩" 패턴:

발명:    오 헨리 단편, 히치콕 초기
모방:    M. 나이트 샤말란 (식스센스, 언브레이커블)
관습:    스릴러 장르에서 반전은 "있어야 하는 것"
클리셰:  모든 스릴러가 반전을 넣으려 함 → 반전 자체가 예측됨
전복:    "노인을 위한 나라는 없다" — 반전 없이 끝남 (반전의 부재가 충격)
복고:    "나이브스 아웃" — 반전을 자각적으로 즐기는 미스터리
```

## 3. Convention vs Cliché 판별 프레임워크

### 5가지 판별 기준

| # | 기준 | Convention | Cliché |
|---|------|-----------|--------|
| 1 | **관객 반응** | "이런 게 있어야지" (만족) | "또 이거야" (지루) |
| 2 | **예측 가능성** | 구조는 예측, 실행은 신선 | 구조도 실행도 예측 |
| 3 | **제거 테스트** | 빼면 이야기 약해짐 | 빼도 이야기 그대로 |
| 4 | **의식 수준** | 알고서 선택 | 무의식적 반복 |
| 5 | **감정 효과** | 여전히 감정 유발 | 감정 없이 "확인란 체크" |

### 실전 자가 테스트 5개

| # | 테스트 | 방법 |
|---|--------|------|
| 1 | **첫 아이디어 테스트** | 가장 먼저 떠오른 것 = 대부분 클리셰. 최소 3개 대안 생성, 가장 예상 못한 것 선택 |
| 2 | **검색 테스트** | TV Tropes에서 검색 가능한가? → 있으면 변형 필요 |
| 3 | **캐릭터 특수성** | 다른 캐릭터에 넣어도 성립 = generic = 클리셰 방향 |
| 4 | **제거 테스트** | 이 요소 빼도 이야기 그대로 = 장식 클리셰 |
| 5 | **관객 예측** | 경험 많은 관객이 다음을 맞출 수 있으면 → 변형 필요 |

## 4. 전복 전략

### 5가지 전복 기법

| 기법 | 메커니즘 | 사례 |
|------|---------|------|
| **해체 (Deconstruction)** | 클리셰를 진지하게 가져감. "실제로 어떻게 될까?" | 왓치맨: 슈퍼히어로가 심리적으로 망가져 있다면? |
| **램프셰이딩 (Lampshading)** | 작품 안에서 클리셰를 인정 | 스크림: "호러 영화에서 살려면 규칙이 있어" |
| **반전 (Inversion)** | 클리셰를 뒤집음 | 슈렉: 오거가 영웅, 미남 왕자가 악당 |
| **재구축 (Reconstruction)** | 해체 후 자각적으로 재건 | 스파이더버스: 영웅 여정을 알면서도 진심으로 |
| **합병증 (Complication)** | 클리셰 쓰되 꼬임 추가 | 장례식 비, 그런데 고인이 비를 좋아했음 → 슬픔이 복잡해짐 |

### 전복의 3가지 전제 조건

```
1. 장르 지식:  전복하려는 관습을 깊이 이해해야 함
              모르고 다르게 하는 것 ≠ 전복 (그건 그냥 "다름")

2. 기대 구축:  관객이 기대를 가져야 전복이 작동
              기대 없으면 전복도 없음 → 관습을 먼저 설립 후 깨기

3. 의미:      전복 자체가 목적이면 "gotcha"일 뿐
              전복이 캐릭터/주제/감정에 봉사해야 진짜 전복
```

### 재구축 사이클 (가장 강력한 형태)

```
관습:     영웅은 항상 이긴다 (슈퍼맨, 1978)
해체:     영웅은 망가지고, 권력은 부패 (왓치맨, 1986)
재구축:   영웅은 망가져 있고 그래도 영웅이 되기로 선택 (스파이더버스, 2018)

재구축 > 원래 관습: 의문을 견뎌내고 진정성으로 나왔기 때문
```

## 7. 핵심 인사이트 요약

```
1. 클리셰는 7가지 차원이 겹친 복합 현상이다.
   관객 기대([1])만 보면 나머지 6개를 놓친다.

2. 클리셰의 긍정적 기능([3] 소통 압축)이 있다.
   특히 150자 프롬프트 제한에서 관습은 필수.

3. 가장 위험한 차원은 [4] 사회/문화.
   관객이 즐겨도 해로울 수 있는 유일한 차원.

4. 같은 패턴이 맥락에 따라 신선/관습/클리셰 전부 될 수 있다.
   절대적 클리셰는 없고, 상대적 판단만 있다.

5. 구조(뼈대)는 클리셰가 될 수 없다([6]).
   클리셰는 항상 특정 실현(살)의 문제.

6. 최고의 전략은 "관습의 효율 + 한 요소 변형".
   100% 신선 = 소통 실패. 100% 관습 = 클리셰.
   한 가지만 틀면 효율적이면서 신선.

7. 재구축(Reconstruction)이 가장 강력한 형태.
   의문을 견뎌낸 진정성 > 원래 관습 > 해체만 한 냉소.
```
