## 1. 모델 정의

영상 제작은 **두 개의 내용 축**(S, V)과 **하나의 메타 레이어**(C)로 구성된다.

```
┌─────────────────────────────────────────────────────┐
│              C (서사 원리 / Narrative Grammar)         │
│                                                     │
│  S에 적용:           S→V 변환:          V에 적용:    │
│  인과, 갈등,         모티프→시각,        편집 리듬,   │
│  CDQ, 핍진성,        템포→컷빈도,       시각 클리셰,  │
│  클리셰 검증         서브텍스트→연기     관객 경험     │
│                                                     │
│         ┌──────┐              ┌──────┐              │
│         │  S   │─── 교차점 ───│  V   │              │
│         │ 내용  │    (16개)    │ 형식  │              │
│         └──────┘              └──────┘              │
│                                                     │
└─────────────────────────────────────────────────────┘
                        │
                   스토리보드 1칸
                  (S + V + C 완전 명세)
```

**셋의 정체**:
- **S (Story)**: 내용 축 -- "무엇을, 왜 말하는가"
- **V (Visual)**: 형식 축 -- "어떻게 보이고 들리는가"
- **C (Craft)**: 메타 레이어 -- "왜 작동하는가, 어떻게 연결되는가"

**핵심 원칙**:
1. S와 V는 직교(orthogonal)한다. 같은 스토리를 다른 비주얼로 만들 수 있고, 같은 비주얼로 다른 스토리를 만들 수 있다.
2. **C는 S, V와 다른 종류이다.** S와 V가 "체스 말과 판"이라면 C는 "체스 규칙"이다. 규칙은 말이나 판과 독립적이 아니라, 말이 판 위에서 어떻게 움직이는지를 지배한다.
3. **최적의 영상은 S와 V가 C의 원리에 의해 서로를 강화할 때** 나온다.

## 2. 두 축의 정체

### S축: Story Layer

| 레벨 | 이름 | 담당 | 변경 비용 |
|------|------|------|----------|
| **S0** | 장르/톤 | 이 영상이 "무엇"인가 | 새 프로젝트 |
| **S1** | 내러티브 구조 | 이야기가 "어떻게" 흘러가는가 | 전체 재구성 |
| **S2** | 캐릭터/관계 | "누가" 이야기를 이끄는가 | 해당 캐릭터 관련 전체 |
| **S3** | 씬/감정/대사 | 각 순간에 "무슨 일"이 일어나는가 | 해당 씬만 |

### V축: Visual Layer

| 레벨 | 이름 | 담당 | 변경 비용 |
|------|------|------|----------|
| **L0** | 매체/포맷 | "어떤 도구"로 만드는가 | 새 프로젝트 |
| **L1** | 비주얼 스타일 | "어떤 느낌"으로 보이는가 | 전체 리메이크 |
| **L2** | 프로덕션 디자인 | "구체적으로 어떤 모습"인가 | 부분 재작업 |
| **L3** | 샷 실행 | "이 순간 카메라/빛/소리"는? | 해당 샷만 |

### 교차점

S의 각 레벨과 V의 각 레벨이 만나는 16개 교차점이 존재.

```
        L0        L1        L2        L3
S0   S0xL0     S0xL1     S0xL2     S0xL3
S1   S1xL0     S1xL1     S1xL2     S1xL3
S2   S2xL0     S2xL1     S2xL2     S2xL3
S3   S3xL0     S3xL1     S3xL2     S3xL3
```

각 교차점의 의미:

| 교차점 | 의미 | 예시 |
|--------|------|------|
| **S0xL0** | 장르가 매체를 선택 | 호러 → 실사(리얼공포) or 2D(심리공포) |
| **S0xL1** | 장르가 스타일을 결정 | 느와르 → 하이콘트라스트, 각진 |
| **S0xL2** | 장르가 디자인 기조 설정 | SF → 미래적 로케이션, 테크 소품 |
| **S0xL3** | 장르가 샷 레시피 결정 | 호러 → 로우앵글, 정적→스팅어 |
| **S1xL1** | 구조가 스타일 변주 요구 | 비선형 → 시간대별 다른 그레이딩 |
| **S1xL3** | POV가 카메라 결정 | 1인칭 → POV샷 중심 |
| **S2xL1** | 캐릭터가 쉐이프 결정 | 악당 성격 → 삼각형 디자인 |
| **S2xL2** | 아크가 의상/컬러 아크 결정 | 타락 → 밝은색→어두운색 |
| **S3xL2** | 씬 감정이 컬러 스크립트 결정 | 절망 → 블루, 차가운 |
| **S3xL3** | 감정 비트가 카메라/조명 결정 | 고백 → 슬로우 돌리 인, 소프트 조명 |

## 3. 양방향 영향 (S ↔ V)

### S → V (스토리가 비주얼을 결정) -- 90% 케이스

일반적 흐름. 스토리가 먼저 존재하고, 그에 맞는 비주얼을 선택.

```
"전쟁 트라우마 이야기" → 포토리얼, 디새추, 핸드헬드, 차가운 팔레트
"장난감 모험 이야기"   → 3D CGI, 밝은, 둥근, 비비드 팔레트
"고독한 우주비행사"    → 와이드 비율, 네거티브 스페이스, 미니멀 사운드
```

### V → S (비주얼이 스토리를 만듦) -- 10%이지만 가장 강력한 순간들

비주얼/기술 결정이 스토리텔링 장치 자체가 되는 경우.

| 사례 | V 결정 | S 효과 |
|------|--------|--------|
| Mommy (돌란) | L0 비율 1:1 → 1.85:1 전환 | 캐릭터가 프레임을 밀어냄 = 희망의 순간 |
| 스파이더버스 | L0 fps: 마일스 on 2s → on 1s | 프레임레이트 변화 = 캐릭터 성장 |
| 브레이킹배드 | L2 컬러 믹싱 | 스카이러 블루 + 제시 옐로 = 월터 그린 |
| 엘사 | L2 의상 전환 | 드레스 체인지 = 캐릭터 해방의 순간 |
| 미야자키 | L3 정적 (마) | 아무 일도 안 일어나는 3초 = 세계의 체험 |
| 디킨스 | L3 구도 빈 공간 | 프레임 80% 비어있음 = 인물의 무의미함 |
| 에드거 라이트 | L3 편집 스매시컷 | 컷 자체가 펀치라인 (코미디) |

### 상호 강화 (S ↔ V 시너지)

최고의 영상은 S와 V가 같은 방향을 가리킬 때:

```
S: "주인공이 거대한 세계에 압도된다"
V: EWS + 네거티브 스페이스 + 미니멀 사운드

  → S와 V가 같은 감정을 전달
  → 효과가 곱해짐 (더하기가 아님)

S: "주인공이 자유를 얻는다"
V: 화면 비율이 넓어짐 + 채도 상승 + 음악 크레센도

  → S의 서사적 자유 + V의 물리적 확장
  → 관객이 "느끼는" 자유
```

반대로 S와 V를 의도적으로 어긋나게 하면 **아이러니/불안**:

```
S: "모든 것이 괜찮다" (대사)
V: 어두운 조명 + 불안한 음악 + 더치 앵글

  → 관객: "거짓말이다. 뭔가 잘못됐다."
  → 서브텍스트를 V가 전달
```

## 4. 동시 협상이 이상적인 이유

```
이상적 워크플로우:

     S축                              V축
     ──                               ──
Step 1:
     S0 장르  ◄══════협상══════►  L0 매체
     "호러를 하고 싶다"             "근데 2D로 하면?"
     "2D 호러는 심리적이네"         "그럼 미니멀 스타일"
     → S0와 L0가 동시에 확정

Step 2:
     S1 구조  ◄══════협상══════►  L1 스타일
     "비선형으로 가자"             "시간대별 다른 스타일?"
     "과거=따뜻한, 현재=차가운"     "그러면 반전 더 효과적"
     → S1에서 비선형 선택이 L1 변주를 열고, L1 변주가 S1 반전을 강화

Step 3:
     S2 캐릭터 ◄═════협상══════►  L2 디자인
     "주인공이 타락한다"           "컬러 아크: 흰→회→검"
     "중간에 한번 돌아오는 건?"     "그러면 잠깐 밝아졌다 다시 어두워지는 거"
     → S2 아크와 L2 컬러가 서로를 조각함

Step 4:
     S3 씬   ◄══════협상══════►  L3 샷
     "여기서 진실을 알게 된다"      "CU + 역광 + 정적"
     "정적이 너무 좋다. 대사를 줄이자" ← V가 S를 수정!
     → L3 프리뷰가 S3 대사를 변경
```

**이 프로세스에서 V→S 피드백이 자연스럽게 발생.**
선형 구조에서는 이 협상을 구조적으로 시뮬레이션해야 함 → `linear_pipeline.md` 참조.

## 5. C 메타 레이어: S와 V의 문법

### C의 정체

C는 S, V와 같은 "축"이 아니다. S와 V가 내용물(체스 말, 체스판)이라면, C는 규칙(체스 규칙)이다.

```
비유:
  S = 체스 말 (킹, 퀸, 폰...)        ← 내용물
  V = 체스판 (8x8 격자, 색상)         ← 표현 형식
  C = 체스 규칙 (이동법, 승리 조건)    ← 작동 원리

  규칙 없이 말과 판만 있으면 = 게임이 아님
  말 없이 규칙과 판만 있으면 = 플레이 불가
  세 가지가 합쳐져야 = 체스
```

### C가 하는 3가지 역할

**역할 1: S를 "이야기"로 만든다 (S 내부 작동)**

| C 카테고리 | S에 대한 역할 | 없으면 |
|-----------|-------------|--------|
| C1 인과 | S3 씬들을 "그래서/그런데"로 연결 | 씬 나열일 뿐, 이야기 아님 |
| C2 극적 엔진 | S 전체를 전진시킴 (Hook, CDQ, 긴장) | 관객이 보는 이유가 없음 |
| C3 전달 기법 | S의 정보를 어떤 순서/속도로 공개할지 결정 | 같은 내용이 지루하거나 혼란 |
| C4 의미 | S 표면 아래 의미 부여 (모티프, 아이러니, 서브텍스트) | 표면적 사건만 남음 |
| C5 관객 설계 | S가 관객에게 어떤 경험이 되는지 설계 | 관객과의 감정 연결 없음 |
| C6 무결성 | S가 믿을 수 있는지 검증 | 핍진성 붕괴 |

**역할 2: S를 V로 변환한다 (S→V 변환 규칙)**

```
S3: "캐릭터가 진실을 발견한다"
     │
     │  C 없이 직접 매핑:
     │  "진실 발견" → ??? (어떤 카메라? 어떤 조명? 기준 없음)
     │
     │  C를 통한 매핑:
     │  C3 전달: "점진적으로 드러낸다"      → L3 느린 돌리 인
     │  C4 모티프: "거울 모티프 사용"       → L2 거울 있는 세트, L3 랙포커스
     │  C5 관객: "관객이 먼저 알게 한다"    → L3 와이드로 단서 보여줌
     │  C4 서브텍스트: "대사와 영상이 다른 것을 말함" → L3 조명 스플릿
     │
     ▼
V:   구체적 L2/L3 결정이 도출됨
```

| C 카테고리 | S→V 변환에서의 역할 |
|-----------|-------------------|
| C3 템포 | S의 감정 비트 → L3 편집 리듬, 샷 길이 |
| C3 정보 은폐 | S의 정보 관리 → L3 카메라/구도로 무엇을 보여주고 숨길지 |
| C4 모티프 | S의 주제 → L2 컬러/소품에 반복 요소 심기 |
| C4 서브텍스트 | S의 텍스트/서브텍스트 갭 → L3 퍼포먼스/조명이 대사와 다른 것 전달 |
| C5 공감 | S의 캐릭터 본딩 → L3 카메라(CU=공감, WS=거리) 선택 |
| C5 서스펜스 | S의 정보 비대칭 → L3 구도(폭탄을 보여줄지 말지) 결정 |

**역할 3: V를 검증한다 (V 품질 보증)**

| C 카테고리 | V에 대한 역할 |
|-----------|-------------|
| C3 템포 | 편집 리듬이 서사 템포와 일치하는지 검증 |
| C4 모티프 | 시각 모티프가 주제에 연결되는지 검증 |
| C6 무결성 | 시각적 연속성, 톤 일관성 검증 |
| 클리셰 | V축 클리셰 감지 (틸앤오렌지 과용, 슬로모 남용 등) |

### C의 6개 카테고리

```
C1 연결 논리     인과 체인, 셋업/페이오프
C2 극적 엔진     갈등 에스컬레이션, 마이크로Hook, 씬브레이크, CDQ
C3 전달 기법     템포 조절, 정보 은폐, 반복/변주, Show vs Tell
C4 의미 층위     주제, 모티프/상징, 아이러니, 서브텍스트
C5 관객 설계     감정 아크, 서스펜스, 공감 메커니즘, 관객 참여, 카타르시스
C6 무결성       핍진성 규칙, 톤 일관성, 세계 규칙 일관성

+ 클리셰 프레임워크 (C 전체를 관통하는 교차 관심사, cliche_framework.md 참조)
```

### 파이프라인에서 C의 작동 지점

```
[S0~S3 완료] → [C 적용 ①] → [Mid Preview] → [C 적용 ②] → [L0~L3] → Video
                   │                             │
              S 내부 검증                     S→V 변환 규칙
              ├ 인과 체인 확인                ├ 모티프→시각 매핑
              ├ CDQ 존재 확인                 ├ 템포→편집 매핑
              ├ 핍진성 검증                   ├ 서브텍스트→연기/조명
              ├ 클리셰 검사                   ├ 정보은폐→카메라/구도
              └ 감정 아크 설계                └ 공감→샷타입 선택
```

## 6. 모델의 수학적 성질

### 변경 비용 비대칭

```
S0 변경 → V 전체 무효화 (100%)
L0 변경 → V 전체 무효화, S는 유지 (V만 100%)

S 변경이 V 변경보다 파급력이 큼:
  S는 "왜"를 바꿈 → V 전체가 따라가야
  V는 "어떻게"를 바꿈 → S는 대부분 유지

따라서 S→V 방향의 의존성이 V→S보다 강함.
이것이 선형 구조(S 먼저)가 합리적인 이유.
```

### 레벨 간 제약 전파

```
상위 → 하위: 강한 제약 (선택지를 좁힘)
하위 → 상위: 약한 피드백 (제안/영감)

S0 ──(강)──> S1 ──(강)──> S2 ──(강)──> S3
│            │            │            │
(강)         (강)         (중)         (중)
│            │            │            │
L0 ──(강)──> L1 ──(강)──> L2 ──(강)──> L3

대각선 제약 (S→V):
S0──(강)──>L0, S0──(강)──>L1, S0──(중)──>L2, S0──(약)──>L3
S2──(중)──>L1, S2──(강)──>L2, S3──(강)──>L3

역방향 (V→S):
L0──(약)──>S0 (매체가 장르 가능성을 제한)
L1──(약)──>S2 (쉐이프가 캐릭터 인상에 영향)
L3──(약)──>S3 (샷 프리뷰가 대사/감정 조정에 영감)
```
