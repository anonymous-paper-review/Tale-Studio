# L0 + L1 딥다이브: 불변/초고비용 변경 요소 전수 조사

> 작성일: 2026-04-15
> 핵심 원칙: "현실적으로 바꿀 수 없는 것"과 "바꾸면 처음부터 다시 만드는 것"을 명확히 구분

---

## PART 1: L0 -- 물리적으로 불변인 결정

한번 정하면 프로덕션 도중 변경이 불가능. 변경 = 전체 프로젝트 폐기 후 재시작.

---

### 1. 영상 매체/방식 (Visual Medium)

가장 근본적인 결정. 이후 모든 L0/L1 결정이 이것에 종속됨.

#### 완전한 분류 체계

**A. 실사 (Live Action)**
- 전통 실사 (물리 세트, 로케이션)
- 버추얼 프로덕션 (LED 월 / StageCraft -- 실제 배우 + 실시간 CG 배경)
- 미니어처/모형 촬영 (강제 원근법, 스케일 모델)
- 수중 촬영
- 항공/드론 촬영

**B. 2D 애니메이션**
- 전통 셀 애니메이션 (물리 셀에 수작업, 촬영)
- 디지털 2D (Toon Boom, TVPaint, Clip Studio -- 벡터 또는 래스터)
- 벡터 기반 (Flash/Animate 레거시, SVG -- 해상도 독립적)
- 컷아웃 애니메이션 (미리 그린 파츠를 디지털 퍼펫 리깅)
- 로토스코프 (실사 위에 프레임 단위 트레이싱 -- A Scanner Darkly, Waking Life)
- 리미티드 애니메이션 (프레임 수 감소, 홀드 포즈 -- 한나-바베라, 웹 애니메이션)
- 풀 애니메이션 (디즈니급, 매 프레임 새로 드로잉)

**C. 3D CGI**
- 포토리얼리스틱 CG (픽사, 드림웍스, 일루미네이션)
- 스타일라이즈드 3D (아케인의 페인트오버 3D, 스파이더버스의 코믹북 3D)
- 리얼타임 3D (언리얼 엔진, 유니티 -- 버추얼 프로덕션 및 최종 출력)
- 볼류메트릭 캡처 (Light Stage, 포토그래메트리)

**D. 스톱모션**
- 퍼펫 애니메이션 (볼앤소켓 아마추어 -- LAIKA: 코렐라인, 쿠보)
- 클레이메이션 (아드만: 월레스와 그로밋, 숀더쉽)
- 종이/컷아웃 스톱모션 (사우스파크 원본 파일럿, 몬티 파이선)
- 레고/브릭 (레고 무비의 CG-모방-스톱모션 하이브리드)
- 실루엣 애니메이션 (로테 라이니거의 아흐메드 왕자)
- 픽실레이션 (실제 인간을 스톱모션 피사체로)
- 오브젝트/샌드/핀스크린 애니메이션

**E. 모션 그래픽**
- 2D 모션 그래픽 (After Effects, 키네틱 타이포그래피)
- 3D 모션 그래픽 (Cinema 4D, Houdini 프로시저럴)
- 데이터 시각화 / 인포그래픽 애니메이션

**F. AI 생성**
- T2V (Text-to-Video: Sora, Veo, Runway, Kling)
- I2V (Image-to-Video)
- V2V (Video-to-Video: 스타일 트랜스퍼)
- 하이브리드 AI 보조 (AI 배경/인비트윈 + 사람 키프레임)

**G. 혼합 매체 (Mixed Media)**
- 실사 + 2D (누가 로저 래빗을 모함했나, 스페이스 잼)
- 실사 + 3D CGI (모던 VFX 영화 거의 전부)
- 2D + 3D 하이브리드 (아케인: 3D+2D 페인트오버, Paperman: 3D+2D 라인 추출)
- 스톱모션 + CGI (쿠보: 실제 퍼펫 + CG 물/이펙트)
- 버추얼 프로덕션 + 프랙티컬 (만달로리안: LED 월 + 물리 세트)
- 로토스코프 + 디지털 페인트 (러빙 빈센트: 매 프레임 유화)

#### 매체 선택 내부의 하위 결정
- **캡처 방식**: 물리 카메라 vs 가상 카메라 vs 스캐너 vs 프로시저럴 생성
- **파이프라인 소프트웨어**: Maya/Houdini/Blender(3D), Toon Boom/TVPaint(2D), Dragonframe(스톱모션), Unreal/Unity(리얼타임)
- **렌더 엔진 락인**: RenderMan vs Arnold vs V-Ray vs Cycles -- 에셋이 한 엔진용으로 만들어지면 마이그레이션 비용 막대

#### 사례 연구

**만달로리안 (버추얼 프로덕션)**: ILM StageCraft -- 6m 높이, 270도 반원형 LED 월, 직경 23m, 언리얼 엔진 구동. 시즌1의 50% 이상이 LED 볼륨에서 촬영. 만달로리안의 베스카 갑옷에 올바른 반사가 자동으로 생김 (그린스크린에서는 불가능). 카메라 패럴랙스에 실시간 대응.

**아케인 (2D-3D 하이브리드)**: Fortiche -- Maya에서 3D 모델 빌드 후 아티스트가 2D 텍스처/이펙트/표정을 3D 렌더 위에 프레임 단위 핸드페인트. 여러 렌더 패스 레이어: 라인워크, 컬러 그래디언트, 섀도, 하이라이트, 그레인. 수작업 조작으로 수채화/유화 느낌 구현. 이펙트(연기, 폭발, 징크스 낙서)는 12fps로 애니메이션 -- 의도적 스타카토.

**러빙 빈센트 (페인티드 로토스코프)**: 매 프레임이 반 고흐 스타일 유화. 실사 배우 촬영 후 125명의 화가가 캔버스에 프레임 단위 핸드페인트. 완전히 새로운 매체 카테고리.

#### 교차 의존성
- 스톱모션 선택 → 쉐이프 언어가 물리적 아마추어/재료로 제한됨
- 2D 전통 선택 → 멀티플레인 없이는 진정한 카메라 패럴랙스 불가
- 실사 선택 → 물리 법칙은 "무료"지만 판타지 요소는 VFX 필수
- AI 생성 → 현재 해상도(1080p~4K), 길이(5~20초), 프레임레이트(24~60fps) 제약

### 2. 해상도 (Resolution)

"몇 픽셀"이 아니라 여러 하위 결정의 묶음.

#### A. 공간 해상도 (픽셀 치수)

| 규격 | 해상도 | 용도 |
|------|--------|------|
| SD NTSC | 720x480 | 레거시 |
| SD PAL | 720x576 | 레거시 |
| HD 720p | 1280x720 | 웹/모바일 |
| Full HD | 1920x1080 | 스트리밍 기본 |
| 2K DCI | 2048x1080 | 디지털 시네마 |
| UHD/4K | 3840x2160 | 소비자 4K |
| 4K DCI | 4096x2160 | 극장 상영 |
| 6K | 6144x3160 | RED Komodo 캡처 |
| 8K | 7680x4320 | 하이엔드 디스플레이 |
| IMAX 디지털 | ~4K~12K | 시스템에 따라 |
| IMAX 15/70 필름 | ~12K~18K 상당 | 최고 해상도 |

#### B. 센서/캡처 사이즈 (실사 -- 스토리보드에서 거의 논의 안 됨)

| 포맷 | 사이즈 | 특성 |
|------|--------|------|
| Micro Four Thirds | 17.3 x 13mm | |
| Super 16 | 12.5 x 7.5mm | 인디 필름 |
| Super 35 | 24.9 x 18.7mm | 가장 일반적 시네마 포맷 |
| Full Frame | 36 x 24mm | 얕은 DOF |
| Large Format | 40.96 x 21.6mm | ARRI Alexa LF, Sony Venice |
| 65mm / IMAX | 52.5~70.4mm | 극단적 디테일 |

센서 사이즈가 DOF, 화각, 저광 성능, 이미지 "느낌" 전체에 영향.

#### C. 비트 뎁스 (색 깊이)

| 비트 | 단계 | 용도 |
|------|------|------|
| 8-bit | 1670만 색 | 소비자 영상, 그래디언트 밴딩 |
| 10-bit | 10.7억 색 | 방송/스트리밍 표준 |
| 12-bit | | 디지털 시네마, RAW 캡처 |
| 16-bit | | VFX 합성, OpenEXR |
| 32-bit float | 무제한 | HDR 합성, 과학 시각화 |

#### D. 크로마 서브샘플링
- **4:4:4** -- 픽셀당 풀 컬러 (RAW, ProRes 4444)
- **4:2:2** -- 수평 컬러 해상도 절반 (방송, ProRes 422)
- **4:2:0** -- 컬러 해상도 1/4 (H.264/H.265 전송, 대부분 스트리밍)

#### E. 색 공간 / 가뭇
- **Rec.709 / sRGB**: 가시 색상의 ~35% -- HD 방송 표준
- **DCI-P3**: ~45% -- 디지털 시네마 상영
- **Rec.2020**: ~75% -- UHD/HDR 타겟
- **ACES (AP0)**: 전체 가시 스펙트럼 포괄 -- 교환 표준

#### F. 다이내믹 레인지
- **SDR**: 디스플레이에서 ~6스톱
- **HDR10**: ~10+스톱, 피크 1000니트
- **Dolby Vision**: ~12+스톱, 피크 4000+니트
- **필름 네거티브**: ~13-14스톱 래티튜드

#### G. 전달 함수 / 감마
- Rec.709 gamma (2.4) / sRGB gamma (~2.2)
- Log 인코딩 (S-Log3, LogC, V-Log, REDLogFilm) -- 캡처 다이내믹 레인지 극대화
- PQ (ST 2084) -- HDR 절대 휘도
- HLG -- 역호환 HDR

#### H. 필름 그레인 / 노이즈 구조
- **필름 그레인**: 유기적, 프레임마다 다른 은염 결정 클러스터
- **디지털 노이즈**: 전자적 센서 아티팩트, 균일한 픽셀 노이즈
- 필름 vs 디지털 촬영은 되돌릴 수 없는 노이즈 특성을 물리적으로 각인

#### 사례 연구

**크리스토퍼 놀란 + IMAX 65mm**: IMAX 15-perf 65mm 필름 = 표준 35mm 대비 ~18배 해상도. 오펜하이머 전편을 라지 포맷 필름으로 촬영.

**핀처 + RED**: 6K+ 촬영 후 후반 리프레이밍 -- 전송 해상도보다 높게 캡처하여 유연성 확보.

**픽사 렌더 해상도**: 극장용 2K 렌더 (4K 업스케일). 코코는 프레임당 초기 렌더 1,000시간 -- 4K로 올리면 4배.

#### 숨겨진 하위 결정
- **오버샘플링 vs 네이티브**: 안정화, 리프레이밍, VFX 추출을 위해 전송보다 높은 해상도로 캡처
- **픽셀 종횡비**: 모든 픽셀이 정사각형은 아님. 아나모픽 캡처는 비정사각형 픽셀
- **렌더 vs 디스플레이 해상도**: 3D에서 캐릭터 풀 해상도, 먼 배경 절반 해상도 등 멀티패스
- **안티앨리어싱 방식**: MSAA, FXAA, TAA, 슈퍼샘플링 -- 에지 품질 결정
- **텍스처 해상도 예산**: UV 타일당 512px~8K -- 서피스 "해상도"는 출력 해상도와 독립

### 3. 프레임 레이트 (Frame Rate)

#### A. 베이스 프레임 레이트

| fps | 용도 |
|-----|------|
| 12 | 리미티드/웹 애니메이션 |
| 16 | 무성영화 시대 표준 (1927 이전) |
| 23.976 | NTSC 호환 "24p" (대부분 디지털 시네마의 실제 fps) |
| 24 | 시네마 표준 (1927 유성영화 이후) |
| 25 | PAL 방송 (유럽, 아시아, 아프리카) |
| 29.97 | NTSC 인터레이스 |
| 30 | NTSC 프로그레시브 |
| 48 | HFR 시네마 (호빗, 아바타 속편) |
| 60 | 게임, 스포츠 방송, 일부 스트리밍 |
| 120 | 제미니맨 (이안) |
| 240+ | 울트라 슬로모션 캡처 (팬텀 카메라) |

#### B. 애니메이션 전용 타이밍 ("on Ns" 시스템)

| 타이밍 | 초당 고유 드로잉 | 느낌 |
|--------|------------------|------|
| **On 1s** | 24 | 최대 유동성, 빠른 액션, 핵심 순간. 노동 집약적 |
| **On 2s** | 12 | 장편 애니메이션 표준. "일러스트" 품질 |
| **On 3s** | 8 | 느린 동작, 리미티드 미학 |
| **On 4s** | 6 | 극단적 리미티드, 아주 느린 앰비언트 |
| **혼합** | 가변 | 대부분 프로 애니메이션: 빠른 액션=1s, 홀드=3s |

**캐릭터별 가변 타이밍 (스토리텔링 장치)**:
스파이더버스에서 마일스 모랄레스는 on 2s (어색, 미숙), 피터 B. 파커는 on 1s (유연, 능숙). 프레임레이트 차이가 캐릭터 아크를 전달.

#### C. 셔터 앵글 / 모션 블러

| 셔터 앵글 | 셔터 속도 (@24fps) | 느낌 |
|-----------|-------------------|------|
| 360도 | 1/24s | 최대 블러, 몽환적 |
| 180도 | 1/48s | 시네마 표준, "영화적" 블러 |
| 90도 | 1/96s | 약간 선명/끊김 |
| 45도 | 1/192s | 매우 선명, 스타카토 (라이언 일병 구하기 전투씬) |

#### D. 인터레이스 vs 프로그레시브
- **프로그레시브 (p)**: 매 프레임이 완전한 이미지. 시네마 표준.
- **인터레이스 (i)**: 홀수/짝수 라인 번갈아 캡처. 빠르게 도태 중.

#### 사례 연구

**스파이더버스 (애니메이션 on 2s as 스토리텔링)**: CG 관례를 깨고 12 고유 포즈/초 (on 2s). 표준 CG의 24 (on 1s) 대신. 결과: 매 프레임이 독립된 일러스트로 읽힘. 캐릭터별 타이밍 차이로 내러티브 의미 전달.

**호빗 48fps**: 피터 잭슨 48fps 촬영. 관객/비평가 분열: "너무 리얼해서 TV 드라마 같다." 모션 블러와 24fps 특유의 몽환적 부드러움이 사라짐. 프레임레이트가 기술 설정이 아니라 **관객과의 미학적 계약**임을 증명.

**일본 애니메이션 타이밍 관례**: 대부분 24fps 제작이지만 대화/평온한 장면은 on 3s (8 드로잉/초), 액션은 on 2s 또는 on 1s. 사쿠가(작화 하이라이트)는 on 1s. 깊이 뿌리박힌 문화적/경제적 관례.

#### 숨겨진 하위 결정
- **스미어 프레임**: On 2s 애니메이션에서 빠른 동작 시 인위적 모션 블러. 왜곡/늘어난 드로잉이 1~2프레임만 존재. 단독으로 보면 추하지만 모션에서 필수.
- **무빙 홀드**: CG 캐릭터가 완전히 멈추면 "죽은" 느낌. 프로들은 미세한 호흡, 무게 이동, 눈 움직임 추가. 이 관례는 어디에도 명시되지 않지만 보편적.
- **립싱크 오프셋**: 프로 관례 -- 입 모양을 오디오보다 1~2프레임 앞에 배치. 시각 처리가 청각보다 느려서 뇌가 동기화함. 명시되지 않지만 보편적으로 실행.
- **롤링 셔터 vs 글로벌 셔터**: 디지털 센서의 읽기 방식. 롤링 셔터 = 빠른 움직임에 "젤리" 흔들림. 카메라 선택에 의해 결정, 스토리보드에서 논의 안 됨.

### 4. 화면 비율 (Aspect Ratio)

#### 완전한 분류

| 비율 | 이름 | 용도 |
|------|------|------|
| 1.00:1 | 정사각형 | 인스타그램 레거시, 실험 영화 |
| 1.19:1 | 무비톤 | 초기 유성영화 (1927-1932) |
| 1.33:1 (4:3) | 아카데미 | 무성영화, 와이드스크린 이전 TV |
| 1.43:1 | IMAX | 풀 IMAX 15/70 프레임 |
| 1.78:1 (16:9) | HDTV | TV 표준, 웹 비디오 기본 |
| 1.85:1 | 미국 플랫 와이드 | 미국/영국 극장 비스코프 이외 표준 |
| 2.00:1 (18:9) | 유니비지움 | 넷플릭스 인기 포맷 |
| 2.39:1 | 스코프/아나모픽 | 현대 아나모픽 와이드스크린 표준 |
| 2.76:1 | Ultra Panavision 70 | 벤허, 헤이트풀 에이트 |
| 0.56:1 (9:16) | 세로 영상 | 모바일 우선, TikTok, Reels, Shorts |
| 가변 | 내러티브 장치 | 영화 중간에 비율 변경 |

#### 아나모픽 vs 스피리컬
- **아나모픽 렌즈**: 더 넓은 화각을 센서에 압축 → 후반에 디스퀴즈. 특유의 타원형 보케, 수평 렌즈 플레어, 가장자리 왜곡.
- **스피리컬 렌즈**: 센서를 크롭하여 와이드스크린 달성. 아나모픽 특유의 시각적 특성 없음.

#### 사례 연구

**웨스 앤더슨의 그랜드 부다페스트 호텔**: 하나의 영화에 세 가지 비율.
- 1.33:1 (아카데미) → 1930년대 시퀀스
- 2.39:1 (스코프) → 1960년대
- 1.85:1 → 현대 프레이밍 스토리

각 비율이 해당 시대의 영화 관례와 일치. 대사나 프로덕션 디자인 전에 시대 전환이 시각적으로 인지됨.

**크리스토퍼 놀란의 IMAX/스코프 전환**: 다크나이트, 인터스텔라, 덩케르크, 테넷, 오펜하이머에서 2.39:1 ↔ 1.43:1 전환. IMAX 시퀀스에서 화면이 "열리는" 물리적 확장감.

**자비에 돌란의 Mommy**: 대부분 1:1 (정사각형). 핵심 희망의 순간에 프레임이 1.85:1로 물리적으로 확장 -- 캐릭터가 손으로 프레임 경계를 밀어냄. **화면 비율 자체가 감정 내러티브**.

**로버트 에거스의 더 라이트하우스**: 1.19:1 (거의 정사각형). 초기 영화 환기 + 등대에 갇힌 두 남자의 밀실 공포감.

#### 숨겨진 하위 결정
- **레터박싱 vs 크롭 전략**: 다른 비율 디스플레이에서 블랙바 vs 크롭. "세이프 영역"이 프레임 어디에.
- **추출 윈도우**: 대형 센서에서 2.39:1 크롭의 정확한 위치 -- 센터/상단/하단 추출.
- **극장 vs 홈 vs 모바일 전송물**: 2.39:1 극장 + 16:9 스트리밍 오픈 매트 + 9:16 모바일 리프레임. 원본 구도가 모든 것을 수용해야.

### 5. 렌더링 방식 (3D 전용)

#### A. 포토리얼리스틱 렌더링
- **패스 트레이싱 / 레이 트레이싱**: 물리적 광전파 시뮬레이션. 비편향. 느림. (RenderMan RIS, Arnold, V-Ray)
- **PBR (물리 기반 렌더링)**: 물리 파라미터로 머티리얼 정의 (알베도, 러프니스, 메탈릭, IOR). 오프라인/리얼타임 모두 표준.

#### B. NPR (비사실적 렌더링)
- **셀/툰 셰이딩**: 평면 컬러 밴드 + 하드에지 라이팅 (보더랜즈, 길티기어, 젤다 야생의 숨결)
- **아웃라인/컨투어**: 에지 디텍션 카툰 외곽선 (인버티드 헐, 소벨 필터, 프리스타일)
- **해칭/크로스해칭**: 펜앤잉크 일러스트 시뮬레이션
- **페인터리**: 3D 지오메트리 위 브러시스트로크 시뮬레이션
- **하프톤**: 벤데이 도트 / 신문 프린트 (스파이더버스의 시그니처)

#### C. 하이브리드
- **스타일라이즈드 PBR**: 물리적 라이팅 + 비사실적 머티리얼 (픽사: 물리 빛, 스타일화된 표면)
- **PBR 위 NPR**: 풀 PBR 렌더 후 합성에서 NPR 적용 (아케인)
- **선택적 NPR**: 리얼 캐릭터 + NPR 환경 또는 반대 (소울: 리얼 NYC vs 추상적 그레이트 비포)

#### D. 리얼타임 렌더링
- **래스터라이제이션 + 스크린스페이스**: 전통 게임 렌더링
- **리얼타임 레이트레이싱 (RTX/DXR)**: HW 가속 (반사, 그림자, GI)
- **나나이트 (언리얼 5)**: 전통 LOD 시스템 제거, 필름급 메시 직접 렌더
- **루멘 (언리얼 5)**: 베이킹 없는 다이내믹 GI. 버추얼 프로덕션에 사용.

#### 렌더 엔진별 생태계

| 엔진 | 특성 |
|------|------|
| RenderMan (픽사) | 업계 표준, SSS/볼류메트릭/헤어 전문 |
| Arnold (오토데스크) | 비편향 패스 트레이서, ILM/MPC/Framestore |
| V-Ray (Chaos) | 바이어스/언바이어스 하이브리드, 건축 시각화 |
| Redshift (Maxon) | GPU 가속, 빠른 이터레이션 |
| Cycles (블렌더) | 오픈소스 패스 트레이서, 인디/미드티어 |
| Karma/Mantra (SideFX) | Houdini 네이티브 |
| Unreal Engine | 리얼타임, 버추얼 프로덕션/프리비즈/최종 픽셀 |

#### 사례 연구

**픽사의 진화**: 토이스토리(1995) REYES(스캔라인, 레이트레이싱 없음) → 플라스틱 룩이 영리한 제약 활용 → 몬스터스 유니버시티(2013) GI 도입 → 코코(2017) 풀 패스 트레이싱. 코코의 죽은 자의 땅: 870만 조명, 2700만 오브젝트, 최적화 전 프레임당 1000시간.

**소울의 이중 렌더링 철학**: 리얼 NYC = 표준 포토리얼 파이프라인 + 고급 SSS + 볼류메트릭 라이팅. 그레이트 비포 = 완전히 새로운 방식: 소울 캐릭터를 볼륨으로 렌더, 프리즘 빛 투과(한쪽 따뜻한, 반대쪽 차가운), "테이퍼드 브러시스트로크" 에지. 하나의 영화에 두 가지 근본적으로 다른 렌더링.

**스파이더버스의 합성 중심 NPR**: 시그니처 룩의 대부분이 3D 렌더러가 아니라 합성(Nuke)에서 생성. 25개 이상 커스텀 툴: "Hatcher"(해치라인), "Thresher"(하프톤 도트). 3D 렌더는 비교적 평범, NPR 변환이 다운스트림에서 발생.

#### 숨겨진 하위 결정
- **SSS 모델**: 스킨 렌더링에서 물리적 볼류메트릭 SSS vs 빠른/페이크 스크린스페이스 SSS
- **헤어/퍼 렌더링**: 스트랜드 기반(수백만 커브) vs 카드 기반(텍스처 폴리곤) vs 볼류메트릭(밀도 필드)
- **천 시뮬레이션 충실도**: 물리 시뮬 vs 스컬프트/포즈 기반 vs 하이브리드
- **GI 방식**: 패스 트레이싱 vs 포톤 매핑 vs 라이트 프로브 vs 스크린스페이스
- **섀도 알고리즘**: 레이트레이스 vs 섀도맵 vs 딥 섀도맵
- **AA 전략**: 픽셀 샘플링 수 (4x, 16x, 64x), 어댑티브 샘플링, 포스트 프로세스 AA

- **코믹스**: 볼드 외곽선, 플랫 컬러, 하프톤 (스파이더버스, 스콧 필그림)

- **유기 추상**: 흐르는 생체형 형태 (소울의 그레이트 비포)

### 7. 쉐이프 언어 (Shape Language)

#### 기본 도형 어휘

| 도형 | 심리적 연상 | 캐릭터 유형 |
|------|------------|------------|
| 원/구 | 따뜻함, 안전, 친근, 순진 | 주인공, 아이, 코믹, 보살피는 인물 |
| 사각/정육면체 | 안정, 신뢰, 힘, 고집 | 수호자, 권위자, 든든한 동료 |
| 삼각 (위) | 역동, 야심, 위계, 공격 | 악당, 야심가, 능동적 주인공 |
| 삼각 (아래) | 불안정, 위험 | 트릭스터, 불안정한 인물 |
| 유기/비정형 | 자연, 유연, 예측 불가 | 자연 정령, 변신 캐릭터 |

#### 적용 레벨
1. **실루엣**: 최외곽 윤곽. 썸네일에서 식별 가능해야 함. 픽사 테스트: 실루엣만으로 캐릭터 식별 가능?
2. **바디 프로포션**: 몸통/머리/팔다리의 기하학적 기반
3. **얼굴 피쳐**: 둥근 눈 vs 각진 눈, 동그란 코 vs 뾰족한 코, 턱 형태
4. **의상**: 각진 갑옷 vs 흐르는 로브 vs 박시 유니폼
5. **소품/탈것**: 악당 아지트=각진, 주인공 집=둥근
6. **환경**: 고담=각진, 샤이어=둥근

#### 프로포션 시스템 (머리-몸 비율)

| 비율 | 느낌 | 사례 |
|------|------|------|
| 1:1~2:1 | 슈퍼 디폼/치비 | 최대 귀여움, 코믹/굿즈 |
| 3:1~4:1 | 카툰 아이 | 클래식 미키마우스, 아동 프로그램 |
| 5:1~6:1 | 표준 카툰 성인 | 심슨스, 밥스 버거 |
| 7:1 | 스타일화된 리얼 | 디즈니 왕자/공주, 대부분 픽사 성인 |
| 7.5:1~8:1 | 리얼리스틱 성인 | 리얼 CG 휴먼, 모캡 캐릭터 |
| 8:1~9:1 | 영웅적 프로포션 | 슈퍼히어로, 패션 일러스트 |
| 10:1+ | 극단적 신장 | 베이오네타 |

#### L0와의 교차 의존성
- **스톱모션**: 물리 아마추어/재료의 제약. 최소 구조적 두께 필요. LAIKA는 3D 프린트 페이스 플레이트로 한계 극복.
- **2D 애니메이션**: 기하학적 자유롭지만 수천 프레임에 걸쳐 여러 아티스트가 일관되게 그릴 수 있어야. 과도하게 복잡한 디자인은 풀 애니메이션에서 유지 불가.
- **3D CGI**: 리깅 복잡도에 의해 제한. 극단적 프로포션은 정교한 디포메이션 시스템 필요.
- **실사**: 캐스팅, 의상, 프로스테틱, 카메라 앵글/렌즈로만 작동. 인간 골격 변형 불가 (CG 보강 없이).

#### 숨겨진 하위 결정
- **탄젠트 회피**: 두 형태/에지가 간신히 닿아 시각적 혼란을 일으키는 것을 본능적으로 회피. 아티스트에게 훈련되지만 브리프에 명시 안 됨.
- **쉐이프 계층**: 모든 디자인에 1차(실루엣), 2차(주요 내부 구분), 3차(디테일) 읽기가 있음. 1차부터 바깥으로 디자인.
- **쉐이프 리듬**: 디자인 전반의 도형 반복과 변주. 원형 캐릭터 = 둥근 단추, 둥근 안경, 통형 몸통.
- **네거티브 스페이스 디자인**: 캐릭터 요소 사이/주변의 빈 공간도 의도적으로 디자인.

### 8. 라인 퀄리티 (Line Quality)

#### A. 라인 웨이트

| 유형 | 특성 | 사례 |
|------|------|------|
| 균일 (모노웨이트) | 전체 동일 두께. 깔끔, 기계적, 그래픽 | 리뉴 클레르: 탱탱 |
| 가변 (씩-씬) | 그림자 쪽 두껍게, 하이라이트 쪽 얇게. 유기적, 서예적 | 디즈니, 지브리 |
| 테이퍼드 | 시작 두껍게 → 끝 얇게 (또는 반대). 역동적 | 많은 애니메이션, 스파이더버스 |
| 가중 계층 | 외곽 실루엣 두껍게, 내부 디테일 얇게 | 보편적 프로 관례 |

#### B. 라인 클린니스

| 유형 | 특성 |
|------|------|
| 클린/기계적 | 완벽히 매끈, 텍스처 없음. 디지털 벡터, 스튜디오 애니메이션 |
| 러프/스케치 | 가시적 컨스트럭션, 여러 패스, 거친 가장자리. 쿠엔틴 블레이크, 인디 애니메이션 |
| 텍스처드 | 도구 고유 텍스처 (챠콜, 연필, 드라이 브러시). 가구야 공주 이야기 |
| 끊어진/암시적 | 시작과 끝이 끊기고 관객의 눈이 형태를 완성 |

#### C. 3D에서의 라인 생성 방식
- **인버티드 헐**: 지오메트리 복제 → 노멀 뒤집기 → 바깥으로 익스트루드 → 검정 셰이딩
- **에지 디텍션 (포스트 프로세스)**: 뎁스/노멀 버퍼에 소벨/라플라시안 필터
- **프리스타일 (블렌더)**: 프로그래머블 라인 시스템
- **Pencil+ (3ds Max)**: 업계 NPR 라인 렌더러
- **핸드드로운 오버레이**: 3D 렌더 위에 프레임당 수작업 (아케인)

#### 스튜디오별 라인 아이덴티티

| 스튜디오 | 특성 |
|----------|------|
| 지브리 | 가변 웨이트, 유기적, 자신감 있는 브러시 스트로크; 그림자 쪽 두껍게; 배경은 더 부드럽고 페인터리 |
| 교토 애니메이션 | 클린, 정밀, 균일~미세 가변; 극도로 정제; 프레임 간 일관 |
| 디즈니 2D | 볼드, 자신감 있는 가변; 제로그래픽 텍스처; 강한 실루엣 |
| 카툰 네트워크 | 매우 두꺼운 외곽선; 최소 웨이트 변화; 강한 그래픽성 |
| 스파이더버스 | 3D 생성 + 프린트 코믹 품질; 하프톤 그리드 스냅 |

#### 교차 의존성
- 라인 퀄리티 + 프레임레이트: on 2s + 두꺼운 가변 라인 ≠ on 1s + 얇은 균일 라인
- 라인 퀄리티 + 해상도: 가는 라인은 높은 해상도 필요 (앨리어싱 방지)
- 라인 퀄리티 + 매체: 진정한 서예적 라인 변화는 수작업 또는 정교한 NPR 필요

### 9. 캐릭터 코어 디자인 (Character Core Design)

#### 기본 프로포션 결정
- 머리-몸 비율, 눈 크기 (리얼리즘 지표), 손 크기 (표현력), 발 크기
- 목 길이/두께, 어깨-엉덩이 비율, 팔다리 길이 비율
- 몸통 길이 vs 다리, 두개골 형태, 얼굴 피쳐 배치 (높은 눈=귀여움, 낮은 눈=성숙/위협)

#### 실루엣 디자인 원칙
- **즉시 가독성**: 단색 실루엣 썸네일에서 캐릭터 식별 가능
- **비대칭**: 미세한 비대칭이 역동성 부여
- **고유 네거티브 스페이스**: 팔다리/바디/소품 사이 빈 공간
- **포스처 = 실루엣**: 기본 자세가 성격 전달 (구부정=게으름, 가슴 펴기=자신감)
- **액세서리 통합**: 모자, 무기, 망토, 헤어 = 영구 실루엣 아이덴티티

#### 일관성 명세 (반드시 필요한 문서)
- 모델 시트 / 턴어라운드: 정면, 3/4, 측면, 후면, 3/4 후면
- 표정 시트: 디자인 프로포션 유지하면서 표정 범위
- 액션/포즈 시트: 극단적 포즈에서 디자인 유지 테스트
- 스케일 차트: 캐릭터 줄 세워 상대적 키/프로포션 비교
- 컬러 모델: 모든 캐릭터 요소의 확정 색상

#### 숨겨진 하위 결정
- **아이 리그 복잡도**: 리얼 눈 = 공막 변형, 홍채 확장, 눈물샘, 눈꺼풀 따라감. 카툰 눈 = 셰이프키 블렌딩.
- **FACS 준수 여부**: 페이셜 리그가 FACS 유닛(AU1~AU46+)에 매핑되는지. 가능한 표정과 조합을 결정.
- **천/머리카락 = 캐릭터 아이덴티티**: 엘사의 브레이드 = 캐릭터 아이덴티티. 모든 샷에서 일관된 시뮬레이션 필수.
- **조인트 수와 디포메이션 해상도**: 필름 캐릭터 얼굴만 300+ 조인트 vs 게임 캐릭터 30개. 한번 결정하면 모든 애니메이션 제약.

### 10. 텍스처 철학 (Texture Philosophy)

#### 서피스 처리 스펙트럼

| 단계 | 특성 | 사례 |
|------|------|------|
| 프리스틴 | 마모 없음, 먼지 없음 | 일루미네이션 (시크릿 라이프 오브 펫) |
| 리브드인/웨더드 | 미세 마모, 먼지, 세월 흔적 | 픽사 (월-E, 토이스토리) |
| 그런지/디스트레스드 | 심한 마모, 녹, 손상 | 포스트 아포칼립틱, 스팀펑크 |
| 스타일화 단순화 | 플랫 컬러, 리얼 텍스처링 없음 | 카툰 네트워크, 모바일 게임 |
| 페인터리 | 텍스처 맵 자체에 가시적 붓터치 | 아케인 핸드페인트 |
| 프로시저럴 | 수학적 생성 (펄린 노이즈, 보로노이) | Houdini 기반 VFX |

#### 텍스처 맵 종류 (기술적)
- **디퓨즈/알베도**: 라이팅 없는 기본 색상
- **노멀 맵**: 픽셀당 서피스 방향 (가짜 마이크로 지오메트리)
- **러프니스/글로시니스**: 표면 매끄러움/거칠기
- **메탈릭**: 금속 여부 분류
- **디스플레이스먼트**: 실제 지오메트리 변형
- **SSS 맵**: 빛이 표면 침투하는 위치와 양 (피부, 왁스, 잎)
- **이미시브**: 자체 발광 (화면, 눈, 마법)
- **AO**: 틈새 사전 연산 그림자
- **이방성**: 방향성 표면 변화 (브러시드 메탈, 모발 광택 방향)

#### 사례 연구

**스파이더버스 텍스처**: 하프톤 넘어 "인쇄된" 텍스처 -- 뉴스프린트 종이 텍스처 오버레이. 다른 유니버스 캐릭터별 다른 텍스처: 그웬=수채화 블리드, 스파이더맨 느와르=모노크롬+헤비 필름 그레인, 페니 파커=클린 플랫 애니메 텍스처.

**아케인 핸드페인트**: 모든 3D 모델의 텍스처를 포토샵에서 수작업 페인팅. 가시적 붓터치, 색 변주, 예술적 의도. 프로시저럴/스캔 텍스처로는 불가능한 룩. 노동 비용 막대하지만 차별화된 결과.

**토이스토리 머티리얼 제약 = 장점**: 플라스틱, 직물, 페인트 나무는 피부/머리카락/자연 재질보다 설득력 있는 텍스처링이 쉬움. 1995년 렌더링 기술의 L0 제약에 의한 L1 결정.

#### 숨겨진 하위 결정
- **UV 레이아웃**: 텍스처 밀도, 심 가시성, 페인팅 워크플로우에 영향. 에셋당 한번 결정.
- **텍스처 필터링**: 바이리니어/트라이리니어/이방성. 비스듬한 각도 텍스처 룩에 영향.
- **머티리얼 ID 시스템**: 머티리얼 태깅 관례. 전체 파이프라인에 영향.
- **BRDF 모델**: 서피스 반사 수학 모델 (GGX, Beckmann, Blinn-Phong). 모든 머티리얼의 스페큘러 하이라이트 변경.
- **텍스처 핸드오프 포맷**: 8-bit PNG vs 16-bit EXR vs TX. 전 에셋의 색 정밀도와 파일 크기.

## PART 3: L0 → L1 교차 의존성 매트릭스

| L0 결정 | 제약하는 L1 | 어떻게 |
|---------|------------|--------|
| **스톱모션** | 아트 스타일 | 물리 재료로 가능한 것으로 제한. "디지털 글로우", 투명 레이어 불가 |
| **스톱모션** | 쉐이프 언어 | 물리 아마추어 수용 필수. 팔다리 최소 두께. 얼굴=조각 또는 교체 플레이트 |
| **스톱모션** | 텍스처 | 텍스처 = 재료 자체. 재건축 없이 "리페인트" 불가 |
| **2D 전통** | 쉐이프 언어 | 수천 프레임 × 여러 아티스트가 일관 드로잉 가능해야 |
| **2D 전통** | 텍스처 | 프레임당 노동. 텍스처 서피스는 매 프레임 재드로잉 또는 합성 |
| **3D CGI** | 아트 스타일 | 기본 출력이 매끈한 "CG룩". 페인터리/스케치/프린트 룩은 의도적 NPR 노력 필요 |
| **3D CGI** | 라인 퀄리티 | 라인은 알고리즘 생성 또는 렌더 위 수작업 필요 |
| **실사** | 쉐이프 언어 | 인간 프로포션 고정. 캐스팅, 의상, 프로스테틱, 카메라로만 작동 |
| **48fps HFR** | 아트 스타일 | 모션 블러 감소 → 모든 것이 "더 리얼". 스타일화/페인터리 접근이 어려워짐 |
| **4:3 비율** | 쉐이프 언어 | 수직 프레이밍 → 수직 캐릭터 구도 유리; 넓은 앙상블 어려움 |
| **포토리얼 렌더** | 아트 스타일 | 물리적 타당성에 고정. 스타일화 컬러/불가능한 조명이 렌더러와 충돌 |
| **NPR 렌더** | 텍스처 | PBR 텍스처 맵이 불필요할 수 있음. 전체 텍스처링 파이프라인 변경 |
| **필름 스톡** | 텍스처 | 그레인 구조가 물리적으로 각인. 이미지 변경 없이 제거 불가 |
| **센서 사이즈** | 모든 시각 속성 | FF vs S35 vs IMAX → DOF, 화각, 렌즈 선택, 이미지 "느낌" 전체 변경 |
| **색 공간** | 아트 스타일 | 넓은 가뭇 = 더 채도 높은 아트 디렉션 가능. P3용 프로덕션은 Rec.709에서 충실 재현 불가 |
| **HDR vs SDR** | 아트 스타일/텍스처 | HDR → 스페큘러/이미시브가 물리적으로 밝음. 하이라이트/반사/광원 아트 디렉션 변경 |

## PART 4: 프로가 암묵적으로 내리는 L0/L1 결정

경험 많은 프로가 "자동으로" 내리지만 AI 파이프라인에서는 명시적으로 캡처해야 하는 결정들.

### 프리프로덕션 암묵 결정
1. **컬러 파이프라인 / 컬러 매니지먼트 (ACES vs 커스텀)**: 에셋 생성 전에 ACES/OCIO 설정 선택. 색상 저장/디스플레이/변환 전체 결정.
2. **네이밍 컨벤션 / 파일 포맷**: TX vs EXR vs PNG (텍스처), ABC vs USD vs FBX (지오메트리). 한번 선택, 모든 도구에 영향.
3. **좌표계와 스케일**: Y-up vs Z-up, 1유닛=1cm vs 1m. 불일치 시 모든 부서에서 연쇄 문제.
4. **스테레오스코픽 3D 결정**: 스테레오 시 눈간 거리, 뎁스 버짓, 수렴 전략 선결 필수.

### 애니메이션 전용 암묵 결정
5. **무빙 홀드**: CG 캐릭터 절대 완전히 멈추면 안 됨. 미세 호흡/무게 이동/눈 움직임 추가. 보편적 관례이나 명시 안 됨.
6. **스미어 프레임 전략**: 빠른 동작의 인위적 블러 접근법 (드로잉 늘리기, 모션 라인, 멀티 익스포저 합성).
7. **립싱크 포니임 오프셋**: 입 모양을 오디오 1~2프레임 앞에. 시각/청각 처리 속도 차이 보정. 명시 안 되지만 보편적.
8. **프레스턴 블레어 포니임 세트**: 립싱크용 표준 8~10개 입 모양(A~X). 구체적 세트 선택이 L1 결정.

### 머티리얼/렌더링 암묵 결정
9. **프레넬 이펙트**: 모든 실제 재질은 비스듬한 각도에서 더 많이 반사. PBR에서는 물리적으로 올바르지만 NPR에서는 포함/억제 결정 필요.
10. **AO 강도와 반경**: 접촉 그림자 얼마나 어둡고 얼마나 퍼지는지. 프로젝트 단위 결정이나 명시 문서화 드묾.
11. **SSS 프로파일**: 피부/왁스/대리석/비취/우유 각각의 산란 거리/색/방향성. 머티리얼 라이브러리당 한번 설정.
12. **폴리곤 예산**: 히어로 캐릭터 50만~200만, 배경 캐릭터 1만~5만, 먼 군중 1천~5천. 초기 결정이 모든 모델링 제약.
13. **LOD 전략**: 에셋 클래스당 디테일 레벨 수. 전환 거리와 방법 (이산 스왑 vs 부드러운 블렌드).

### 전송 암묵 결정
14. **세이프 프레임**: 타이틀 세이프 / 액션 세이프 -- 다른 디스플레이 크롭에서 살아남을 핵심 시각 정보 영역. 프로는 자동으로 이 마진 내에서 구도.
