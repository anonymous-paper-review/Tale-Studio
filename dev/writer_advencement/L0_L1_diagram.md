## 1. 전체 구조 개요

```
                ┌──────────────────┴──────────────────┐

        ┌───────▼───────┐                     ┌───────▼───────┐

        │  (변경 불가)    │                    │ (변경=리메이크)  │

                │                                     │
```

```
           ║

           ║    페인터리/스케치/프린트 = 의도적 NPR 노력
           ║

           ║    진정한 서예적 가변 = 자동 달성 어려움
           ║
```

## 5. L1 내부 상호 의존성

```
L1 요소들 사이의 양방향 영향

    [S] 아트 스타일 ◄══════════════════════════════════════╗
         │                                                ║

         │  (리얼 = 인간 프로포션 / 카툰 = 자유)        ║     ║
         ▼                                           ║     ║
    [C] 캐릭터 코어 ─────────────────────────────╗   ║     ║
         │  프로포션이 리깅을 결정                  ║   ║     ║
         │  리깅이 가능한 표정/동작을 결정          ║   ║     ║
         ▼                                      ║   ║     ║
    [L] 라인 퀄리티 ◄═══════════════════════════╝   ║     ║
         │  캐릭터 복잡도가 라인 스타일을 제약        ║     ║
         │  (복잡한 디자인 + 가변라인 = 작업량 폭증)  ║     ║
         ▼                                         ║     ║
    [T] 텍스처 철학 ◄══════════════════════════════╝     ║
         │  쉐이프 언어가 텍스처 접근을 결정                ║
         │  (각진 = 하드에지 텍스처 / 둥근 = 소프트)       ║
         │                                                ║
         └──── 텍스처가 아트 스타일의 최종 표현을 결정 ═════╝
                (페인터리 텍스처 = 페인터리 스타일 강화)


                    === 순환 의존성 ===

    ┌────> [S] 아트 스타일 ────> [H] 쉐이프 ────> [C] 캐릭터 ───┐

    │                                                          │
    └──────────────────────────────────────────────────────────┘

    실무에서는 이 순환을 "이터레이션"으로 해결:
    스타일 초안 → 캐릭터 시도 → 스타일 조정 → 캐릭터 조정 → ... → 확정
```
