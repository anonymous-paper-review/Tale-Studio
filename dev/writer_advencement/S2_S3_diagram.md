## 2. S0 → S1 → S2 → S3 제약 흐름도

```
S0 ══제약══> S1

[G] 장르 ═════════════════════════════════════════════════════════════
     |
     ├══> [ST] 구조유형: 호러=서서히끓는+반전 / 로맨스=만남→장애→합일
     ├══> [PV] POV: 미스터리=제한시점 / 에픽=전지적 / 호러=1인칭
     └══> [CL] 클라이맥스: 액션=물리적대결 / 드라마=감정적결정

[T] 톤 ══════════════════════════════════════════════════════════════
     |
     ├══> [TH] 주제: 어두운톤=상실/부패/진실 / 밝은톤=성장/사랑/우정
     └══> [CL] 클라이맥스: 어두운=피로스승리/비극 / 밝은=완전한해결

───────────────────────────────────────────────────────────────────

S1 ══제약══> S2

[ST] 구조유형 ════════════════════════════════════════════════════════
     |
     ├══> [AC] 아크: 3막=변화아크최적 / 순환구조=순환아크 / 비선형=다중시점
     ├══> [AR] 아키타입: 영웅여정=전통아키타입 / 앙상블=분산아키타입
     └══> [RL] 관계맵: 삼각구조=3인갈등 / 여정구조=멘토-학생

[PV] POV ════════════════════════════════════════════════════════════
     |
     ├══> [PS] 심리: 1인칭=주인공내면집중 / 전지적=다수캐릭터내면
     ├══> [VO] 보이스: 1인칭=나레이션보이스 / 3인칭=대사중심
     └══> [BH] 행동패턴: 제한시점=캐릭터정보제한 / 전지적=관객에모든정보

[TH] 주제/모티프 ═════════════════════════════════════════════════════
     |
     ├══> [PS] 심리: 주제가캐릭터의want/need결정
     |         "자유" → want=탈출, need=자기수용
     |         "복수" → want=보복, need=해방/용서
     └══> [AC] 아크: 주제가아크방향결정
                     "구원" → 구원아크 / "타락" → 부정적아크

───────────────────────────────────────────────────────────────────

S2 ══제약══> S3

[AR] 아키타입 ════════════════════════════════════════════════════════
     |
     ├══> [PU] 씬목적: hero=갈등/결정씬중심 / mentor=설정/설치씬
     └══> [RD] 관계표현: 아키타입조합이씬다이나믹결정

[AC] 아크 ═══════════════════════════════════════════════════════════
     |
     ├══> [EM] 감정비트: 아크단계가감정범위제약
     |         Act1=설정감정 / Midpoint=전환감정 / Act3=해소감정
     ├══> [PU] 씬목적: 아크비트가특정씬목적요구
     |         암흑의순간=갈등+발견씬 / 진실수용=변환씬
     └══> [CF] 캐릭터핏: 아크진행단계가행동허용범위결정

[PS] 심리 ═══════════════════════════════════════════════════════════
     |
     ├══> [DL] 대사: want/need이대사의서브텍스트결정
     ├══> [CF] 캐릭터핏: 결정스타일이씬내행동제약
     |         충동적=빠른결정 / 분석적=숙고씬필요
     └══> [EM] 감정비트: 감정처리방식이반응패턴결정
              외향적=표현적리액션 / 내향적=억제된리액션

[VO] 보이스 ═════════════════════════════════════════════════════════
     |
     ├══> [DL] 대사: 어휘/리듬/직접성이대사스타일결정
     └══> [CF] 캐릭터핏: 스트레스시보이스변화패턴

[RL] 관계맵 ═════════════════════════════════════════════════════════
     |
     ├══> [RD] 관계표현: 권력다이나믹이씬스테이징결정
     ├══> [DL] 대사: 관계역사가서브텍스트레이어생성
     └══> [IF] 정보관리: 관계가정보공유패턴결정
              신뢰=정보공유 / 불신=정보은폐

[BH] 행동패턴 ═══════════════════════════════════════════════════════
     |
     ├══> [CF] 캐릭터핏: 행동일관성검증의기준
     └══> [EM] 감정비트: 성격별사건처리차이
```

## 3. S2 내부 교차 의존성 매트릭스

```
           AR     AC     PS     VO     RL     BH
AR 아키타입 --     >>>    >>     >      >>     >>
AC 아크     <<<    --     >>>    >>     >>     >>>
PS 심리     <<     <<<    --     >>>    >>     >>>
VO 보이스   <      <<     <<<    --     >>     >>
RL 관계맵   <<     <<     <<     <<     --     >>
BH 행동패턴 <<     <<<    <<<    <<     <<     --

>>> = 강한 제약 (아크가 행동패턴을 지배)
>>  = 중간 제약
>   = 약한 제약
<<< = 역방향 강한 영향
```

**최고 커플링 쌍:**
1. **심리 <-> 행동패턴**: wound/lie가 행동의 직접적 원천
2. **아크 <-> 행동패턴**: 아크 단계가 행동 허용범위 결정
3. **아크 <-> 심리**: Lie→Truth 여정이 want/need 충돌의 본체
4. **심리 <-> 보이스**: 내면 상태가 말하는 방식을 결정
5. **관계맵 <-> 보이스**: 관계에 따라 레지스터 전환

## 4. S3 내부 교차 의존성 (씬 단위 실시간 상호작용)

```
S3 요소들의 상호 영향 -- 하나를 바꾸면 다른 것도 조정 필요

[PU] 씬목적 ◄────────────────────────────────────────────────────┐
  |                                                              |
  ├──> [EM] 감정비트: 목적이 감정 범위 결정 (갈등=고강도, 전환=저강도)  |
  ├──> [DL] 대사: 목적이 대사 기능 결정 (설정=정보, 갈등=충돌)       |
  ├──> [IF] 정보관리: 목적이 정보 공개 시점 결정                     |
  └──> [CF] 캐릭터핏: 목적이 캐릭터 행동 방향 제약                   |
                                                                 |
[EM] 감정비트 ───────────────────────────────────────────────────|
  |                                                              |
  ├──> [DL] 대사: 감정이 대사 톤/속도/서브텍스트 결정               |
  ├──> [CF] 캐릭터핏: 감정 변화가 행동 일관성 검증 필요              |
  └──> [RD] 관계표현: 감정 상태가 관계 상호작용 패턴 영향            |
                                                                 |
[IF] 정보관리 ──> [EM] 감정비트: 정보 공개가 감정 전환점 트리거      |
              ──> [DL] 대사: 정보 비대칭이 서브텍스트 생성            |
              ──> [PU] 씬목적: 정보 타이밍이 씬 기능 결정 ──────────┘

[DL] 대사 ────> [CF] 캐릭터핏: 대사가 캐릭터 일관성의 1차 검증 대상
          ────> [RD] 관계표현: 대사 패턴이 관계 권력 표현

[CF] 캐릭터핏 ────> [EM] 감정비트: 성격이 감정 처리 방식 결정
              ────> [DL] 대사: 보이스 일관성 검증

[RD] 관계표현 ────> [DL] 대사: 관계 역사가 서브텍스트 레이어 생성
              ────> [CF] 캐릭터핏: 관계가 행동 선택지 제약
```

## 5. S2xS3 교차점: 핍진성 매트릭스

```
S2 요소가 S3 요소를 제약하는 16개 교차점

              PU씬목적    EM감정비트   DL대사     IF정보     CF캐릭터핏  RD관계표현
AR 아키타입    >>         >           >          ~         >>         >>
              아키타입이   아키타입이    아키타입    ~         아키타입의   아키타입 조합이
              씬유형결정   감정범위제약  톤에영향              역할기대     관계유형결정

AC 아크       >>>        >>>         >>         >>        >>>        >>
              아크비트가   아크단계가    아크이     아크이     아크진행이   아크가 관계
              씬목적요구   감정궤적결정  대사변화유발 정보공개   행동허용범위  아크도 영향
                                                 타이밍영향  결정

PS 심리       >>         >>>         >>>        >>        >>>        >>
              wound가     감정처리     want/need   정보에대한  결정논리    관계에서
              씬필요성유발  방식결정     서브텍스트   캐릭터반응  핵심결정     역할패턴

VO 보이스     ~          >>          >>>        >         >>         >>
              ~          감정이보이스  직접적      정보전달    보이스일관성  관계에따라
                         변화유발     스타일결정   방식영향    검증        레지스터전환

RL 관계맵     >>         >>          >>         >>>       >>         >>>
              관계갈등이   관계가감정   관계역사가   신뢰/불신이  관계가      직접적으로
              씬발생유발   트리거       서브텍스트   정보흐름결정  행동제약     관계표현

BH 행동패턴   >          >>          >>         >         >>>        >>
              행동이씬     행동이감정   행동스타일   행동이      일관성의     행동이관계
              전개영향     반응결정     이대사영향   정보수집에   근본검증     역학에영향

>>> = 강한 제약 (위반 시 핍진성 붕괴)
>>  = 중간 제약 (위반 시 어색함)
>   = 약한 제약 (위반 시 경미한 불일치)

              ══> [L3-DL] 딜리버리
                  emotion+delivery → 음성연기지시

## 7. 완전한 Dual-Axis 교차 매트릭스 (S2/S3 x L2/L3)

```
S2xS3가 V축과 만나는 핵심 교차점 (총 24개 중 가장 중요한 10개)

교차점            의미                    예시
──────           ─────                   ─────
S2[AC] x L2[CM]  아크가 컬러 아크 결정     타락=밝→어둡 / 구원=어둡→밝
S2[AC] x L2[WD]  아크가 의상 아크 결정     성장=캐주얼→리더 / 타락=단정→파괴
S2[PS] x L3[PF]  심리가 퍼포먼스 결정      wound=미세표정+습관 / 스트레스=신체반응
S2[VO] x L3[DL]  보이스가 딜리버리 결정     blunt=빠른/강한 / indirect=느린/우회
S2[RL] x L3[CO]  관계가 프레이밍 결정       지배=위/크게 / 복종=아래/작게
S3[PU] x L3[CA]  씬목적이 카메라레시피결정  갈등=OTS빠른교차 / 변환=슬로우돌리
S3[EM] x L3[LT]  감정이 조명 결정          고강도=8:1하드 / 저강도=1:1소프트
S3[EM] x L2[CP]  감정이 컬러 결정          절망=블루뮤트 / 희망=워밍비비드
S3[IF] x L3[CA]  정보관리가 카메라 결정     아이러니=WS+CU대비 / 서스펜스=제한시야
S3[CF] x L3[PF]  캐릭터핏이 퍼포먼스 검증   일관성위반=의도적이면강력/비의도면핍진성붕괴
```

## 8. 핍진성 붕괴 위험 지점 (Red Flags)

```
자동 검출 가능한 핍진성 위반:

CRITICAL (즉시 수정 필요):
  [!] 캐릭터 결정이 psychology.values와 정면 충돌 (아크 비트 없이)
  [!] 보이스 프로필과 대사 스타일 불일치 (탈보이스 사유 없이)
  [!] 아크 비트 누락 (긍정적변화인데 암흑의순간 없음)
  [!] 감정 변화가 물리적 불가능 (1초 내 슬픔→기쁨, 촉매 없이)
  [!] Idiot Ball (더 명백한 대안 존재, 선택하지 않은 이유 없음)

WARNING (검토 필요):
  [?] 감정 강도가 3 씬 연속 동일 (관객 피로 위험)
  [?] 관계 아크 전환점이 갑작스러움 (축적 부족 가능)
  [?] 캐릭터가 2 씬 이상 대사 없음 (존재 이유 재검토)
  [?] 정보 비대칭이 3 씬 이상 해소되지 않음 (관객 좌절 위험)
  [?] 그룹 씬에서 캐릭터 역할 불명확 (누가 왜 여기 있는가)

SUGGESTION (품질 향상):
  [~] 포일 관계 미활용 (대비 캐릭터 배치로 깊이 증가 가능)
  [~] 서브텍스트 부재 (모든 대사가 표면적)
  [~] 감정 마스킹 미사용 (캐릭터 복잡성 증가 기회)

## 9. 현재 타입 시스템 대비 확장

```
현재 src/types/scene.ts:

  Character = {
    characterId, name, role,
    description, fixedPrompt, referenceImages
  }
  Scene = {
    sceneId, act, narrativeSummary, originalTextQuote,
    location, timeOfDay, mood, charactersPresent,
    estimatedDurationSeconds
  }

제안 확장 (S2+S3):

  Character → CharacterSpec
    + archetype (primary, secondary, modern_variant)
    + arc (type, start_state, end_state, key_beats)
    + psychology (wound, lie, want, need, motivation_level, values)
    + voice (vocabulary, sentence_length, directness, pace, etc.)
    + behavior (decision_style, stress_response, body_language)

  Scene → SceneSpec
    + purpose (primary, secondary)
    + emotion (entry, turn, exit, intensity, escalation_pattern)
    + information (mode, audience_knows, character_knows, revealed, setup, payoff)
    + dialogue (has_dialogue, key_lines, subtext_note, function_priority)
    + character_scene_fit (per-character believability notes)

  (신규) RelationshipSpec
    + character pair + type + power_dynamic + arc + subtext + history

  (신규) BelievabilityCheck
    + 자동 검증 규칙 셋 (CRITICAL / WARNING / SUGGESTION)
```
